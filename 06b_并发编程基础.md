## 1. 协程与异步编程


> **🤔 为什么需要异步编程？**
> 网络请求要等1秒，这1秒 CPU 在干嘛？闲着！异步让你在等待时去做别的事——"等外卖的时候先扫地"。
>
> **🎯 类比理解**
> - 同步 = 排队买奶茶（前面的人没买完你就得等）
> - 异步 = 下单后拿号去逛街，做好了叫你（等待时不阻塞）
> - asyncio = 一个超级快的服务员，在多个顾客间来回切换，哪个准备好了就服务谁
>
> **💡 设计思路**
> 异步的核心是**协作式多任务**：
> - 每个 `await` 是一个"让出控制权"的点
> - 事件循环负责调度：谁准备好了就运行谁
> - 一个线程就能处理成千上万的并发连接（不像多线程那样消耗资源）
>
> 适用场景：**IO 密集型**（网络请求、文件读写、数据库查询）。CPU 密集型不适合（因为没有"等待"可以利用）。
>
> **🔄 举一反三**
> - Web 框架（FastAPI、aiohttp）基于 asyncio
> - JavaScript 的 async/await 是同一概念
> - 爬虫同时请求100个网页 → asyncio 比多线程更高效
> - 思考：asyncio 跟多线程的区别？asyncio 在每个 `await` 点协作式让出——**两个 await 之间的代码是原子的**，不会被打断；但**跨 await 的"读-改-写"仍可能竞争**（其他协程可能在让出期间也读到旧值），需要 `asyncio.Lock` 或避免在临界区里 await。"asyncio 没有竞态条件" 是常见误解。

### 基础用法

```python
import asyncio

async def fetch(name, delay):
    print(f"[{name}] 开始...")
    await asyncio.sleep(delay)  # 非阻塞等待
    print(f"[{name}] 完成!")
    return f"{name}_data"

async def main():
    # 并发执行3个任务
    results = await asyncio.gather(
        fetch("API-1", 2),
        fetch("API-2", 1),
        fetch("API-3", 3),
    )
    print(results)  # 总耗时 ≈ 3秒（不是2+1+3=6秒）

asyncio.run(main())
```

### 异步迭代与异步上下文

```python
# 异步 for 循环
async for item in async_generator():
    process(item)

# 异步 with 语句
async with AsyncDB() as conn:
    result = await conn.query("SELECT ...")
```

### Task 与取消

> **💡 思路：Task 是对协程的包装，可以取消、获取状态**

```python
async def main():
    task = asyncio.create_task(long_running())
    await asyncio.sleep(1)
    task.cancel()  # 取消！
    try:
        await task
    except asyncio.CancelledError:
        print("任务已取消")
```

### 信号量（限制并发）

> **💡 思路：不是所有资源都能无限并发（如数据库连接池），用信号量控制**

```python
sem = asyncio.Semaphore(5)  # 最多5个并发

async def limited_fetch(url):
    async with sem:  # 超过5个就排队
        return await real_fetch(url)
```

## 5. 描述符 Descriptor

> **🤔 为什么需要描述符？**
> 当多个类的多个属性需要相同的验证逻辑时（如"必须为正数"、"必须为字符串"），描述符让你把验证逻辑定义一次，到处复用。
>
> **🎯 类比理解**
> 描述符 = 智能表单字段。你定义一个"必须是正整数的字段模板"，然后在多个表单上复用它。每次填写时自动验证。
>
> **💡 设计思路**
> 描述符协议：定义了 `__get__`/`__set__`/`__delete__` 的类。当它作为**类属性**时，Python 会在属性访问时自动调用这些方法。
>
> 本质是**拦截属性访问**——在"读取/设置属性"这个动作上加钩子。property 底层就是描述符。
>
> **🔄 举一反三**
> - ORM 框架的字段类型（Django 的 IntegerField、CharField）就是描述符
> - property 是最简单的描述符
> - 描述符是 Python 属性访问机制的底层基础

```python
class Positive:
    """确保属性值为正数的描述符"""
    def __set_name__(self, owner, name):
        self.name = f"_{name}"
    
    def __get__(self, obj, objtype=None):
        if obj is None: return self
        return getattr(obj, self.name, None)
    
    def __set__(self, obj, value):
        if value <= 0:
            raise ValueError(f"必须为正数, 得到 {value}")
        setattr(obj, self.name, value)

class Product:
    price = Positive()   # 复用！
    quantity = Positive() # 复用！
    
    def __init__(self, price, quantity):
        self.price = price
        self.quantity = quantity

p = Product(10, 5)   # ✅
# Product(-1, 5)     # ❌ ValueError
```

## 2. 并发编程基础

> **🤔 为什么需要并发？**
> 单核时代已过去。要充分利用硬件、要处理大量IO、要让程序"同时做多件事"——都需要并发。
>
> **🎯 类比理解**
> - 多线程 = 一个厨师快速在多个灶台间切换（共享厨房，可能撞到）
> - 多进程 = 多个厨师各自在独立厨房工作（隔离安全，但开销大）
> - 异步 = 一个超级快的服务员（轮流服务，不需要多人）
>
> **💡 设计思路（Python 的 GIL）**
> Python 有 GIL（全局解释器锁），意味着**多线程不能真正并行执行 Python 代码**。
> - IO 密集型 → 多线程/asyncio（等待时释放 GIL）
> - CPU 密集型 → 多进程（绕过 GIL，真正并行）
>
> **🔄 举一反三**
> - 下载100个文件 → ThreadPoolExecutor 或 asyncio
> - 计算密集的数据处理 → ProcessPoolExecutor
> - Web 服务器 → asyncio（FastAPI）或多进程（gunicorn）

### 选择指南

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 网络请求/文件IO | asyncio 或 ThreadPool | IO等待时切换 |
| CPU计算（数学/加密） | ProcessPool | 绕过 GIL |
| 简单后台任务 | threading | 实现简单 |
| 高并发Web服务 | asyncio | 资源消耗最少 |

### concurrent.futures（最推荐的并发接口）

> **💡 思路：统一的接口，换一个词就能切换线程/进程**

```python
from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor

# IO密集：线程池
with ThreadPoolExecutor(max_workers=10) as executor:
    results = list(executor.map(download, urls))

# CPU密集：进程池（接口完全一样！）
with ProcessPoolExecutor(max_workers=4) as executor:
    results = list(executor.map(heavy_compute, data))
```

### 线程安全

> **💡 思路：多线程共享数据时必须加锁，否则会出现竞态条件**

```python
import threading

lock = threading.Lock()
counter = 0

def safe_increment():
    global counter
    with lock:  # 加锁保护共享资源
        counter += 1
```
