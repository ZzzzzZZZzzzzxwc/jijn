## 1. 生成器 Generator


> **🤔 为什么需要生成器？**
> 如果你有1亿条数据，全加载到列表里内存就爆了。生成器是**"边算边给"**——每次只产出一个值，用完即弃，几乎不占内存。
>
> **🎯 类比理解**
> 列表 = 把整本书打印出来交给你（一次性占空间）
> 生成器 = 说书人，你说"下一个"，他才讲下一段（按需生产）
>
> 也像自来水管：水龙头（next）开一下出一点水（yield），不是把整个水库搬到你家。
>
> **💡 设计思路**
> 生成器的核心是**惰性求值**（Lazy Evaluation）：
> - 不预先计算所有结果
> - 调用 next() 时才计算下一个
> - yield 让函数"暂停并记住状态"
>
> 这解决了"数据量大但每次只用一个"的场景。是空间复杂度从 O(n) 降到 O(1) 的利器。
>
> **🔄 举一反三**
> - 读取大文件 → 逐行 yield，不 readlines()
> - 数据管道 → 生成器串联，像 Unix 管道 `cat | grep | sort`
> - 无限序列 → 只有生成器能表示（列表做不到）
> - 思考：数据库游标、分页查询本质都是"惰性获取"

### 基本用法

```python
def count_up(n):
    i = 1
    while i <= n:
        yield i  # 暂停，返回值，下次从这里继续
        i += 1

gen = count_up(5)
next(gen)  # 1
next(gen)  # 2（从上次 yield 处继续）

for num in count_up(5):
    print(num)  # 1, 2, 3, 4, 5
```

### 生成器 vs 列表：内存对比

```python
import sys
squares_list = [x**2 for x in range(1000000)]  # ~8MB
squares_gen = (x**2 for x in range(1000000))   # ~120 bytes!
```

### 无限生成器

> **💡 思路：生成器可以表示"无穷"——用列表做不到**

```python
def fibonacci():
    a, b = 0, 1
    while True:  # 永远不停
        yield a
        a, b = b, a + b

from itertools import islice
list(islice(fibonacci(), 10))  # 取前10个
```

### yield from（委托）

> **💡 思路：把子生成器的产出"透传"给调用者**

```python
def flatten(nested):
    for item in nested:
        if isinstance(item, list):
            yield from flatten(item)  # 委托给递归调用
        else:
            yield item

list(flatten([1, [2, 3], [4, [5, 6]]]))  # [1,2,3,4,5,6]
```

### 管道模式（生成器串联）

> **💡 思路：每个生成器做一步处理，像流水线工人**

```python
def read_lines(path):
    with open(path) as f:
        for line in f:
            yield line.strip()

def grep(pattern, lines):
    for line in lines:
        if pattern in line:
            yield line

def upper(lines):
    for line in lines:
        yield line.upper()

# 组装管道：读取 → 过滤 → 转换
pipeline = upper(grep("ERROR", read_lines("log.txt")))
for error in pipeline:
    print(error)
```

### 🔬 生成器底层机制：frame 暂停与恢复

> **💡 生成器不是"语法糖"——它是一个真正的状态机。**
> 理解这一层，才能理解 `send()`/`throw()`/`close()`、`yield from`、以及 asyncio 的基石。

#### 生成器的四种状态

```python
import inspect

def gen_func():
    yield 1
    yield 2

g = gen_func()
print(inspect.getgeneratorstate(g))   # GEN_CREATED  ← 刚创建，未执行

next(g)
print(inspect.getgeneratorstate(g))   # GEN_SUSPENDED ← yield 处暂停

next(g)
try:
    next(g)
except StopIteration:
    pass
print(inspect.getgeneratorstate(g))   # GEN_CLOSED   ← 执行完毕
```

| 状态 | 含义 | 转换条件 |
|------|------|----------|
| `GEN_CREATED` | 创建但未启动 | `next()` 或 `send(None)` → SUSPENDED |
| `GEN_RUNNING` | 正在执行（只有生成器自身能看到） | yield → SUSPENDED |
| `GEN_SUSPENDED` | 在 yield 处暂停 | `next()`/`send()`/`throw()` → RUNNING |
| `GEN_CLOSED` | 执行完毕或被 close() | 终态，不可恢复 |

#### 🔬 字节码视角：yield 做了什么

```python
import dis

def gen():
    x = yield 1
    yield x + 2

dis.dis(gen)
```

```
  2           0 RESUME                   0
              2 LOAD_CONST               1 (1)
              4 YIELD_VALUE              1        ← 暂停，把 1 返回给调用者
              6 RESUME                   1        ← 恢复时从这里继续
              8 STORE_FAST               0 (x)   ← send() 传入的值存到 x

  3          10 LOAD_FAST                0 (x)
             12 LOAD_CONST               2 (2)
             14 BINARY_OP                0 (+)
             18 YIELD_VALUE              1
             20 RESUME                   1
             22 POP_TOP
             24 RETURN_CONST             0 (None)
```

**关键**：
- `YIELD_VALUE`：把栈顶值返回给 `next()`/`send()` 的调用者，**冻结当前 frame**
- `RESUME`：恢复执行。`send(val)` 把 val 压入栈顶，所以 `x = yield 1` 中 `x` 接收的是 send 传入的值
- 生成器的 frame **不销毁**——它保存在 `gen.gi_frame` 中，所有局部变量都还在

#### send()：双向通信

```python
def accumulator():
    """接收外部值并累加"""
    total = 0
    while True:
        value = yield total    # ← 产出 total，接收外部 send 的值
        if value is None:
            break
        total += value

gen = accumulator()
next(gen)           # 启动（必须先 next 或 send(None)，走到第一个 yield）
                    # 返回: 0

gen.send(10)        # 返回: 10  （total=10）
gen.send(20)        # 返回: 30  （total=30）
gen.send(5)         # 返回: 35
```

**执行流程**：
```
next(gen)    → 执行到 yield total → 返回 0, 暂停
gen.send(10) → value=10, total=10 → 执行到 yield total → 返回 10, 暂停
gen.send(20) → value=20, total=30 → 执行到 yield total → 返回 30, 暂停
```

> ⚠️ **第一次必须用 `next()` 或 `send(None)`**。因为第一次执行还没到 yield，没有"接收位"。

#### throw()：向生成器注入异常

```python
def gen_with_cleanup():
    try:
        while True:
            yield "working"
    except GeneratorExit:
        print("正常关闭")
    except ValueError as e:
        print(f"收到异常: {e}")
        yield "recovered"

g = gen_with_cleanup()
next(g)                        # 'working'
result = g.throw(ValueError("oops"))  # 收到异常: oops
print(result)                  # 'recovered'
```

**用途**：协程中用于取消任务、传递错误。asyncio 的 `Task.cancel()` 底层就是 `coro.throw(CancelledError)`。

#### close()：优雅关闭

```python
g = gen_with_cleanup()
next(g)
g.close()   # 向生成器注入 GeneratorExit → 触发 finally/except 清理
# 输出: 正常关闭
```

#### yield from 的完整语义（不只是转发！）

```python
# 表面看：yield from sub_gen = for item in sub_gen: yield item
# 实际上：yield from 还透传 send() 和 throw()！

def sub():
    """子生成器"""
    received = yield "from sub"
    return f"sub got: {received}"    # ← return 值通过 StopIteration.value 传回

def main():
    """委托生成器"""
    result = yield from sub()        # ← 等待子生成器完成，获取 return 值
    print(f"result = {result}")

g = main()
print(next(g))        # 'from sub'   ← 透传子生成器的 yield
print(g.send("hi"))   # 触发: result = sub got: hi
                      # 然后 main() 走完 → StopIteration
```

**yield from 做了什么（伪代码）**：
```python
# result = yield from sub_gen 等价于：
_iter = iter(sub_gen)
_value = next(_iter)
while True:
    try:
        _sent = yield _value           # 透传产出值
    except GeneratorExit:
        _iter.close()
        raise
    except BaseException as _e:
        _value = _iter.throw(_e)       # 透传异常
    else:
        try:
            _value = _iter.send(_sent) # 透传 send 值
        except StopIteration as _e:
            result = _e.value          # 获取 return 值
            break
```

→ 这就是为什么 `yield from` 是 asyncio 协程的基础——它实现了**完整的双向通信透传**。

#### 生成器 vs 协程 的关系

```
Python 演进路线：
├── 生成器 (yield)           → 单向产出
├── 增强生成器 (send/throw)  → 双向通信
├── yield from              → 委托 + 透传
├── @asyncio.coroutine + yield from  → 旧式协程
└── async def + await       → 原生协程（语法糖，底层仍是生成器机制）
```

> **💡 asyncio 的 await 底层就是 yield from 的变体**。理解了生成器的 frame 暂停/恢复机制，异步编程就不再神秘。

## 2. 迭代器 Iterator

> **🤔 为什么需要理解迭代器协议？**
> Python 的 for 循环、解包、列表推导...底层全是迭代器协议在工作。理解它让你能自定义"可遍历的对象"。
>
> **🎯 类比理解**
> - 可迭代对象 (Iterable) = 一本书（有 `__iter__`：能给你翻书的手指）
> - 迭代器 (Iterator) = 翻书的手指（有 `__next__`：知道当前位置，能翻到下一页）
>
> **💡 设计思路**
> 迭代器协议只需要两个方法：
> - `__iter__()` → 返回迭代器自身
> - `__next__()` → 返回下一个值，结束时抛 StopIteration
>
> for 循环的本质：`for x in obj` = 先调 `iter(obj)` 拿迭代器，再不断调 `next()` 直到 StopIteration。
>
> **🔄 举一反三**
> - 自定义数据结构想支持 for 循环？实现 `__iter__`
> - 注意：迭代器是一次性的（用完就空了），可迭代对象可以多次迭代

```python
class CountDown:
    def __init__(self, start):
        self.current = start
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        self.current -= 1
        return self.current + 1

for n in CountDown(5):
    print(n)  # 5, 4, 3, 2, 1
```

### 可迭代对象 vs 迭代器

```python
lst = [1, 2, 3]
# lst 是可迭代对象，不是迭代器
# next(lst)  ❌

it = iter(lst)  # 获取迭代器
next(it)  # 1
next(it)  # 2
```

> **💡 最佳实践：让 `__iter__` 返回新迭代器（用 yield），这样对象可以重复遍历**

```python
class MyRange:
    def __init__(self, stop):
        self.stop = stop
    
    def __iter__(self):
        current = 0
        while current < self.stop:
            yield current  # 每次调用 __iter__ 创建新的生成器
            current += 1

r = MyRange(3)
list(r)  # [0, 1, 2]
list(r)  # [0, 1, 2] — 可以重复遍历！
```

```python
class Range:
```

### 练习 4（综合）

实现"扁平化嵌套列表"生成器。

<details><summary>答案</summary>

```python
def flatten(nested):
    for item in nested:
        if isinstance(item, list):
            yield from flatten(item)
        else:
            yield item

# [1, 2, 3, 4, 5, 6]
```
