## 11.2 多线程：threading

### 11.2.1 基本用法

```python
import threading, time

def worker(name, delay):
    for i in range(3):
        time.sleep(delay)
        print(f"[{name}] step {i}")

t1 = threading.Thread(target=worker, args=("A", 0.5))
t2 = threading.Thread(target=worker, args=("B", 0.3))

t1.start()       # 启动线程
t2.start()
t1.join()        # 等待 t1 结束
t2.join()
print("done")
```

### 11.2.2 ⚠️ 共享状态的危险

```python
counter = 0

def incr():
    global counter
    for _ in range(100_000):
        counter += 1     # 非原子！

threads = [threading.Thread(target=incr) for _ in range(5)]
for t in threads: t.start()
for t in threads: t.join()

print(counter)
# 期望: 500_000
# 实际: 通常 < 500_000  (典型 ~470_000)
```

### 11.2.3 ★ 同步原语

#### Lock（互斥锁）

```python
import threading

counter = 0
lock = threading.Lock()

def incr():
    global counter
    for _ in range(100_000):
        with lock:           # 自动 acquire/release
            counter += 1

# ... 启动 5 个线程 ...
print(counter)   # 输出: 500_000  ✅
```

**Lock 性质**：
- 一次只能一个线程持有
- 同一线程**不能重复 acquire**（会死锁）

#### RLock（可重入锁）

```python
rlock = threading.RLock()

def outer():
    with rlock:
        inner()

def inner():
    with rlock:    # ✅ 同一线程可以再次 acquire
        do_work()
```

适用于**递归调用**或**多个方法都加锁**的类。

#### Semaphore（信号量）

```python
# 限制并发数（如最多 5 个连接）
sem = threading.Semaphore(5)

def request():
    with sem:
        time.sleep(1)

threads = [threading.Thread(target=request) for _ in range(20)]
# 同时最多只有 5 个 request 在执行
```

#### Event（事件信号）

```python
ready = threading.Event()

def consumer():
    print("等待数据...")
    ready.wait()        # 阻塞，直到 ready.set()
    print("拿到数据")

def producer():
    time.sleep(2)
    ready.set()         # 通知所有等待者

threading.Thread(target=consumer).start()
threading.Thread(target=producer).start()
```

#### Condition（条件变量）

```python
buf = []
cond = threading.Condition()

def producer():
    while True:
        with cond:
            buf.append(item := get_item())
            cond.notify()        # 通知一个消费者

def consumer():
    while True:
        with cond:
            while not buf:
                cond.wait()      # 释放锁等待
            item = buf.pop(0)
        process(item)
```

### 11.2.4 ⚠️ 死锁

```python
lock_a = threading.Lock()
lock_b = threading.Lock()

def task1():
    with lock_a:
        time.sleep(0.1)
        with lock_b:    # 如果 task2 已持有 b 等 a → 死锁
            pass

def task2():
    with lock_b:
        time.sleep(0.1)
        with lock_a:
            pass
```

**避免方法**：
1. 总是按**固定顺序**获取多个锁
2. 使用 `lock.acquire(timeout=...)`
3. 尽量减少持锁范围
4. 用更高层抽象（队列）代替手动锁

### 11.2.5 队列：推荐的线程通信方式

```python
import queue, threading

q = queue.Queue(maxsize=10)    # 内置线程安全

def producer():
    for i in range(100):
        q.put(i)               # 满时阻塞
    q.put(None)                # 哨兵：通知消费者结束

def consumer():
    while True:
        item = q.get()         # 空时阻塞
        if item is None:
            break
        process(item)
        q.task_done()

threading.Thread(target=producer).start()
threading.Thread(target=consumer).start()
```

**Queue 类型**：
- `queue.Queue`：FIFO
- `queue.LifoQueue`：LIFO（栈）
- `queue.PriorityQueue`：优先队列

## 11.3 多进程：multiprocessing

### 11.3.1 基本用法

```python
from multiprocessing import Process

def cpu_task(n):
    s = 0
    for i in range(n):
        s += i * i
    return s

if __name__ == "__main__":      # 必须有这个保护！
    p1 = Process(target=cpu_task, args=(10**7,))
    p2 = Process(target=cpu_task, args=(10**7,))
    p1.start(); p2.start()
    p1.join(); p2.join()
```

### 11.3.2 ⚠️ 为什么必须 `if __name__ == "__main__":`

multiprocessing 在 Windows/macOS 上默认用 **spawn** 启动子进程，子进程会重新导入主模块。如果没有保护，子进程会再次创建 Process → 无限递归。

### 11.3.2.1 ★★★ fork-after-thread：Linux 生产事故高发点

🔬 **CPython 在 Linux 上默认 start_method 仍是 `fork`**（直到 3.14 才改默认为 forkserver；3.12 已发出 DeprecationWarning）。fork() 与多线程结合是 **生产环境最危险的组合之一**。

```python
# ❌ 致命陷阱
import threading, multiprocessing, logging

logger = logging.getLogger()  # logger 内部有锁

def worker(): logger.info("hi")     # 持有 logger 锁

# 父进程已经启动了一些线程（如 logging handler、grpc、连接池心跳）
threading.Thread(target=worker, daemon=True).start()

# 此时 fork() 一个子进程
# fork 只复制调用 fork 的那个线程，不复制其他线程
# 但子进程继承了 *锁的状态*——某些锁可能正被父进程的另一个线程持有
p = multiprocessing.Process(target=worker)
p.start()    # 子进程启动 → 调用 logger.info → 等锁 → 死锁，永远不返回
```

**症状**：随机的子进程 hang、`celery` worker 启动后无响应、`gunicorn` 偶发 worker 死锁。

**强烈推荐**：**显式设置 spawn**（macOS 14+ 已改为默认 spawn）：

```python
import multiprocessing
if __name__ == "__main__":
    multiprocessing.set_start_method('spawn')   # 安全：完全独立的新进程
    # 或 'forkserver'：fork 一个干净的服务进程做模板
```

| 方法 | 速度 | 安全性 | 平台 |
|---|---|---|---|
| fork | 最快 | ⚠️ fork-after-thread 死锁风险 | Linux 默认 |
| spawn | 慢（重启 Python） | 安全 | Windows/macOS 14+ 默认 |
| forkserver | 中等 | 安全 | Linux 可选，推荐生产 |

**额外陷阱**：
- fork 后子进程文件描述符是 dup 的——同一个 socket/db connection **不能两边用**
- `requests.Session` / `psycopg2 connection` / `redis.Redis` 都不可在 fork 后共享
- 解决：在 worker 启动时 **重建** 连接

### 11.3.3 进程间通信

#### Queue（进程安全）

```python
from multiprocessing import Process, Queue

def worker(q):
    q.put({"result": 42})

if __name__ == "__main__":
    q = Queue()
    p = Process(target=worker, args=(q,))
    p.start(); p.join()
    print(q.get())     # {'result': 42}
```

#### Pipe

```python
from multiprocessing import Process, Pipe

def child(conn):
    conn.send("hello")
    conn.close()

if __name__ == "__main__":
    parent_conn, child_conn = Pipe()
    p = Process(target=child, args=(child_conn,))
    p.start()
    print(parent_conn.recv())   # 'hello'
    p.join()
```

#### 共享内存

```python
from multiprocessing import Process, Value, Array

def worker(num, arr):
    num.value = 42
    arr[0] = 99

if __name__ == "__main__":
    n = Value('i', 0)            # int
    a = Array('i', [0,0,0])      # int 数组
    p = Process(target=worker, args=(n, a))
    p.start(); p.join()
    print(n.value, a[:])         # 42 [99, 0, 0]
```

#### Manager（高级共享）

```python
from multiprocessing import Process, Manager

def worker(d, lst):
    d["key"] = "value"
    lst.append(1)

if __name__ == "__main__":
    with Manager() as m:
        d = m.dict()
        lst = m.list()
        p = Process(target=worker, args=(d, lst))
        p.start(); p.join()
        print(dict(d), list(lst))
```

### 11.3.4 ⚠️ 进程间数据通过 pickle 序列化

```python
# 不能传不可 pickle 的对象
# 例如 lambda、文件句柄、socket、生成器

# ❌ 这会失败
Process(target=lambda: None).start()
```

## 11.4 ★ concurrent.futures（强烈推荐）

`concurrent.futures` 提供**统一接口**，线程池和进程池只差一个名字。

### 11.4.1 ThreadPoolExecutor

```python
from concurrent.futures import ThreadPoolExecutor
import requests

def fetch(url):
    return requests.get(url).status_code

urls = ["https://example.com"] * 10

with ThreadPoolExecutor(max_workers=5) as ex:
    results = list(ex.map(fetch, urls))
    
print(results)
```

### 11.4.2 ProcessPoolExecutor

```python
from concurrent.futures import ProcessPoolExecutor

def cpu_task(n):
    return sum(i*i for i in range(n))

if __name__ == "__main__":
    with ProcessPoolExecutor(max_workers=4) as ex:
        results = list(ex.map(cpu_task, [10**7]*4))
```

### 11.4.3 submit + as_completed

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

with ThreadPoolExecutor(max_workers=5) as ex:
    futures = [ex.submit(fetch, url) for url in urls]
    
    for future in as_completed(futures):    # 哪个先完成先处理
        try:
            result = future.result(timeout=10)
            print(result)
        except Exception as e:
            print(f"failed: {e}")
```

### 11.4.4 异常传播

```python
def may_fail(x):
    if x == 5:
        raise ValueError("oops")
    return x * 2

with ThreadPoolExecutor() as ex:
    futures = [ex.submit(may_fail, i) for i in range(10)]
    for f in futures:
        try:
            print(f.result())
        except ValueError as e:
            print(f"error: {e}")
```

## 11.5 常见并发模式

### 模式 1：生产者-消费者

```python
import threading, queue

q = queue.Queue(maxsize=10)
DONE = object()

def producer():
    for i in range(100):
        q.put(item := generate(i))
    q.put(DONE)

def consumer():
    while True:
        item = q.get()
        if item is DONE:
            break
        process(item)

threading.Thread(target=producer).start()
threading.Thread(target=consumer).start()
```

### 模式 2：扇出-扇入（map-reduce）

```python
from concurrent.futures import ProcessPoolExecutor

def map_phase(chunk):
    return sum(chunk)

def reduce_phase(partial_sums):
    return sum(partial_sums)

if __name__ == "__main__":
    data = list(range(10**7))
    chunks = [data[i::4] for i in range(4)]
    
    with ProcessPoolExecutor(max_workers=4) as ex:
        partials = list(ex.map(map_phase, chunks))
    
    total = reduce_phase(partials)
```

### 模式 3：限速消费

```python
import threading, time

sem = threading.Semaphore(10)    # 最多 10 个并发请求

def request(url):
    with sem:
        time.sleep(1)            # 模拟请求
        return ...
```

### 模式 4：超时与取消

```python
from concurrent.futures import ThreadPoolExecutor, TimeoutError

with ThreadPoolExecutor() as ex:
    f = ex.submit(slow_task)
    try:
        result = f.result(timeout=5)
    except TimeoutError:
        f.cancel()        # 注意：已开始执行的 future 不能真正取消
        print("超时")
```

⚠️ Python 没有可靠的"杀死线程"机制。`cancel()` 只能取消还没开始的任务。

## 11.6 实际开发场景

### 场景 1：并发下载

```python
from concurrent.futures import ThreadPoolExecutor
import requests

def download(url, path):
    r = requests.get(url, timeout=30)
    with open(path, "wb") as f:
        f.write(r.content)
    return path

urls = [...]
with ThreadPoolExecutor(max_workers=20) as ex:
    futures = {ex.submit(download, url, f"/tmp/{i}.bin"): url 
               for i, url in enumerate(urls)}
    
    for f in as_completed(futures):
        try:
            path = f.result()
            print(f"saved {path}")
        except Exception as e:
            url = futures[f]
            print(f"failed {url}: {e}")
```

### 场景 2：CPU 密集图像处理

```python
from concurrent.futures import ProcessPoolExecutor
from PIL import Image

def thumbnail(path):
    img = Image.open(path)
    img.thumbnail((128, 128))
    out = path.replace(".jpg", "_thumb.jpg")
    img.save(out)
    return out

if __name__ == "__main__":
    paths = [...]
    with ProcessPoolExecutor() as ex:
        thumbs = list(ex.map(thumbnail, paths))
```

### 场景 3：背景任务

```python
import threading

def background_log_uploader():
    while True:
        upload_logs()
        time.sleep(60)

t = threading.Thread(target=background_log_uploader, daemon=True)
t.start()
# daemon=True：主程序退出时该线程自动结束
```

## 11.7 常见误区

### 误区 1：CPU 密集用多线程

```python
# ❌ 多线程算 fibonacci 不会更快
threads = [Thread(target=fib, args=(35,)) for _ in range(4)]
```

应该用 `ProcessPoolExecutor` 或 NumPy。

### 误区 2：忘记 join

```python
# ❌
t = Thread(target=task)
t.start()
print("done")    # 主线程退出时子线程可能未完成
```

### 误区 3：在子进程使用全局状态

```python
# 子进程是独立的内存空间，修改不影响父进程
shared = []

def worker():
    shared.append(1)    # 子进程的副本

Process(target=worker).start()
print(shared)    # [] —— 父进程没变化
```

要用 `Queue` / `Manager` / `shared memory`。

### 误区 4：以为 `daemon=True` 能优雅清理

```python
# daemon 线程是被强制终止，不是优雅退出
# 数据库连接、文件句柄可能没关闭
```

### 误区 5：Pool 的开销没考虑

```python
# 如果每个任务只跑 1ms，进程创建+IPC 远大于计算时间
# 此时不要用进程池
```

## 11.8 面试常问

### Q1：threading 和 multiprocessing 的区别？

**答**：
- threading：同进程多个线程，共享内存，受 GIL 限制
- multiprocessing：多个进程，独立内存，绕过 GIL，但 IPC 有开销

### Q2：什么是死锁？怎么避免？

**答**：多个线程互相持有对方需要的锁，永远等待。避免：
1. 固定顺序获取锁
2. 用 timeout
3. 减少持锁范围
4. 用更高层抽象（Queue）

### Q3：daemon 线程和普通线程的区别？

**答**：daemon 线程在主线程退出时被强制结束；普通线程会阻止程序退出。日志、心跳等后台任务用 daemon。

### Q4：concurrent.futures 和 threading 的区别？

**答**：concurrent.futures 是**高级抽象**，提供 Future、统一接口（线程/进程切换只改一个词）、自动管理线程池。threading 是底层 API。**优先用 concurrent.futures**。

### Q5：如何在线程间通信？

**答**：
- 共享变量 + Lock（不推荐）
- queue.Queue（推荐）
- Event / Condition（信号通知）
- Future（任务结果）

## 11.9 练习题

### 练习 11.1（基础）

实现一个并发的下载器，给定 URL 列表，使用线程池下载，返回结果列表（按完成顺序）。

<details>
<summary>答案</summary>

```python
from concurrent.futures import ThreadPoolExecutor, as_completed
import requests

def download_all(urls, max_workers=10):
    results = []
    with ThreadPoolExecutor(max_workers=max_workers) as ex:
        futures = {ex.submit(requests.get, url): url for url in urls}
        for f in as_completed(futures):
            url = futures[f]
            try:
                results.append((url, f.result().status_code))
            except Exception as e:
                results.append((url, str(e)))
    return results
```
</details>

### 练习 11.2（中级）

实现线程安全的 LRU 缓存：

<details>
<summary>答案</summary>

```python
import threading
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = OrderedDict()
        self._lock = threading.Lock()
    
    def get(self, key):
        with self._lock:
            if key not in self.cache:
                return None
            self.cache.move_to_end(key)
            return self.cache[key]
    
    def put(self, key, value):
        with self._lock:
            if key in self.cache:
                self.cache.move_to_end(key)
            self.cache[key] = value
            if len(self.cache) > self.capacity:
                self.cache.popitem(last=False)
```
</details>

### 练习 11.3（高级）

用 ProcessPoolExecutor 实现并行 word count：

<details>
<summary>答案</summary>

```python
from concurrent.futures import ProcessPoolExecutor
from collections import Counter

def count_chunk(text):
    return Counter(text.lower().split())

def parallel_word_count(text, n_workers=4):
    # ⚠️ 不能简单按字节切——会把 "hello" 切成 "hel"+"lo"，伪词被错误计入
    # 必须在切点向前找空白，保证切割边界落在词与词之间
    cuts = [0]
    for i in range(1, n_workers):
        pos = len(text) * i // n_workers
        while pos < len(text) and not text[pos].isspace():
            pos += 1
        cuts.append(pos)
    cuts.append(len(text))
    chunks = [text[cuts[i]:cuts[i+1]] for i in range(n_workers)]

    with ProcessPoolExecutor(max_workers=n_workers) as ex:
        partials = list(ex.map(count_chunk, chunks))

    total = Counter()
    for p in partials:
        total.update(p)
    return total
```

> 🔬 **真实生产事故的微缩版**：MapReduce / 分块统计任何按字节切割的实现都会遇到这个问题（边界切碎了一个 token）。生产里常见解法：(1) 按行切（`splitlines`）；(2) 按 token 边界切；(3) 切完后把每块"边角余料"传给下一块拼接。
</details>
