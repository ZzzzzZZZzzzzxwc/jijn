## 12.1 从生成器理解协程

### 12.1.1 生成器的"暂停-恢复"能力

```python
def gen():
    print("step 1")
    yield 1
    print("step 2")
    yield 2
    print("step 3")

g = gen()
print(next(g))    # step 1 → 1
print(next(g))    # step 2 → 2
print(next(g))    # step 3 → StopIteration
```

`yield` 的真正含义：**暂停函数，把控制权交给调用者**。

### 12.1.2 协程：能接收值的生成器

```python
def coro():
    while True:
        x = yield                # 接收值
        print(f"got {x}")

c = coro()
next(c)            # 启动，运行到第一个 yield
c.send("hello")    # 给 yield 表达式赋值，并恢复运行
c.send("world")
```

### 12.1.3 ★ async/await：协程的现代语法

旧式（Python 3.4）：

```python
import asyncio

@asyncio.coroutine          # 已废弃
def old_style():
    yield from asyncio.sleep(1)
```

新式（Python 3.5+）：

```python
async def new_style():
    await asyncio.sleep(1)
```

**async def** 定义协程函数，**await** 等价于 `yield from`（带特殊语义）。

## 12.2 协程对象：调用 ≠ 执行

### 12.2.1 ⚠️ 调用 async 函数不会运行它

```python
async def hello():
    print("hello")

result = hello()    # 没有打印！
print(type(result)) # 输出: <class 'coroutine'>
print(result)       # 输出: <coroutine object hello at 0x...>
```

**关键认知**：调用 `hello()` 只**创建协程对象**，不执行。

### 12.2.2 ★ 三种运行协程的方式

```python
import asyncio

# 方式 1：asyncio.run（最常用，程序入口）
async def main():
    print("hello")

asyncio.run(main())

# 方式 2：在已运行的事件循环中 await
async def outer():
    await main()

# 方式 3：作为 Task 调度
async def main():
    task = asyncio.create_task(some_coro())
    await task
```

## 12.3 ★ 事件循环：一切的核心

### 12.3.1 事件循环的工作模型（伪代码）

```python
def event_loop():
    while tasks_or_io_pending():
        # 1. 检查 I/O：哪些 fd 就绪了？
        ready_fds = epoll_wait(timeout=...)
        for fd in ready_fds:
            wake_up_corresponding_coroutine(fd)
        
        # 2. 处理就绪的 task
        while ready_tasks:
            task = ready_tasks.pop()
            try:
                task.coroutine.send(None)    # 推进协程到下一个 await
            except StopIteration:
                task.done = True
        
        # 3. 处理定时器（asyncio.sleep 等）
        for timer in due_timers():
            wake_up_corresponding_coroutine(timer)
```

### 12.3.2 await 的执行流程

```python
async def fetch(url):
    print(f"开始 {url}")
    response = await get(url)        # ① 让出控制权
    print(f"完成 {url}")
    return response
```

**`await get(url)` 的执行**：
1. 调用 `get(url)` 返回一个 awaitable（coroutine 或 Future）
2. 协程**暂停**，把自己注册到事件循环
3. 事件循环去跑其他 task
4. 当 `get(url)` 完成（如收到 HTTP 响应），事件循环唤醒该协程
5. 协程从 `await` 处**恢复**，得到结果

### 12.3.3 ★★★ 关键：await 不"阻塞线程"，只"暂停协程"

```python
async def task(name, delay):
    print(f"[{name}] start")
    await asyncio.sleep(delay)       # 暂停 delay 秒（协程暂停，不是线程睡）
    print(f"[{name}] end")

async def main():
    await asyncio.gather(
        task("A", 2),
        task("B", 1),
        task("C", 3),
    )

import time
start = time.time()
asyncio.run(main())
print(f"total: {time.time() - start:.1f}s")

# 输出：
# [A] start
# [B] start
# [C] start
# [B] end       ← 1秒后
# [A] end       ← 2秒后
# [C] end       ← 3秒后
# total: 3.0s   ← 不是 6 秒！
```

## 12.4 Task 与 Future

### 12.4.1 Task = 调度中的协程

```python
async def main():
    # await 直接执行（顺序）
    result1 = await coro1()
    result2 = await coro2()
    
    # create_task：放入事件循环并发执行
    t1 = asyncio.create_task(coro1())
    t2 = asyncio.create_task(coro2())
    result1 = await t1
    result2 = await t2
```

**区别**：
- `await coro()`：等当前协程跑完才继续
- `create_task(coro())`：让协程**立即开始调度**，可在之后任何时机 await

### 12.4.2 ★ asyncio.gather：并发等待多个

```python
async def main():
    results = await asyncio.gather(
        fetch("https://a.com"),
        fetch("https://b.com"),
        fetch("https://c.com"),
    )
    # 所有都完成才返回
```

**返回**：与输入顺序一致的结果列表。

**行为**：
- 默认任一抛异常 → gather 抛出，但其他 task 仍在运行
- `return_exceptions=True`：异常作为结果返回，不中断

### 12.4.3 asyncio.wait：更细粒度

```python
done, pending = await asyncio.wait(
    [task1, task2, task3],
    timeout=5,
    return_when=asyncio.FIRST_COMPLETED,
)
# 第一个完成就返回，其他还在 pending
```

### 12.4.4 Future：底层的"等待中的值"

Task 是 Future 的子类。Future 是更底层的"未完成的结果"：

```python
loop = asyncio.get_event_loop()
future = loop.create_future()

async def callback():
    await asyncio.sleep(1)
    future.set_result("done")

asyncio.create_task(callback())
result = await future            # 输出: 'done'
```

通常你不直接用 Future，但理解它有助于看懂源码。

## 12.5 取消与超时

### 12.5.1 取消 Task

```python
async def long_running():
    try:
        await asyncio.sleep(100)
    except asyncio.CancelledError:
        print("被取消，做清理")
        raise            # 必须重新抛出（除非你真要吞掉）

async def main():
    task = asyncio.create_task(long_running())
    await asyncio.sleep(1)
    task.cancel()
    try:
        await task
    except asyncio.CancelledError:
        print("确认取消")
```

### 12.5.2 超时

```python
# Python 3.11+ 推荐
async def main():
    try:
        async with asyncio.timeout(5):
            await long_running()
    except asyncio.TimeoutError:
        print("超时")

# 旧式
async def main():
    try:
        result = await asyncio.wait_for(long_running(), timeout=5)
    except asyncio.TimeoutError:
        print("超时")
```

## 12.6 异步上下文管理器与异步迭代器

### 12.6.1 async with

```python
class AsyncResource:
    async def __aenter__(self):
        await self.connect()
        return self
    
    async def __aexit__(self, exc_type, exc_val, tb):
        await self.close()

async def main():
    async with AsyncResource() as r:
        await r.do_work()
```

实际场景：`aiohttp.ClientSession()`、`aiopg.connect()`。

### 12.6.2 async for

```python
class AsyncRange:
    def __init__(self, n):
        self.n = n
        self.i = 0
    
    def __aiter__(self):
        return self
    
    async def __anext__(self):
        if self.i >= self.n:
            raise StopAsyncIteration
        await asyncio.sleep(0.1)
        self.i += 1
        return self.i - 1

async def main():
    async for i in AsyncRange(3):
        print(i)
```

### 12.6.3 异步生成器

```python
async def stream(url):
    async with session.get(url) as resp:
        async for line in resp.content:
            yield line.decode()

async def main():
    async for line in stream("https://..."):
        process(line)
```

## 12.7 ⚠️ 常见陷阱

### 陷阱 1：忘记 await

```python
async def fetch(url):
    return await get(url)

async def main():
    fetch("https://...")    # ❌ 协程对象被丢弃，不会执行
    
    # 还会得到警告：
    # RuntimeWarning: coroutine 'fetch' was never awaited
```

✅ 正确：
```python
await fetch("https://...")
# 或
asyncio.create_task(fetch("https://..."))
```

### 陷阱 2：在 async 中调用阻塞代码

```python
async def bad():
    time.sleep(1)              # ❌ 阻塞整个事件循环
    requests.get(url)          # ❌ 同步 I/O，阻塞

async def good():
    await asyncio.sleep(1)     # ✅
    async with aiohttp.ClientSession() as s:
        async with s.get(url) as r:    # ✅
            return await r.text()
```

**绝对禁止**：在 async 函数里直接用 `time.sleep`、`requests`、同步文件读写。

### 陷阱 3：CPU 密集任务

```python
async def cpu_task():
    return sum(i*i for i in range(10**8))   # ❌ 阻塞事件循环数秒

# ✅ 扔给线程池
async def good():
    loop = asyncio.get_running_loop()
    return await loop.run_in_executor(None, cpu_task_sync)
```

### 陷阱 4：在多个事件循环间共享 Task

每个事件循环只能跑自己的 Task。不能跨循环共享。

### 陷阱 5：忘记关闭资源

```python
# ❌
session = aiohttp.ClientSession()
# ... 用完忘记 await session.close()

# ✅ 用 async with
async with aiohttp.ClientSession() as session:
    ...
```

## 12.8 实战：HTTP 客户端

```python
import asyncio
import aiohttp

async def fetch(session, url):
    async with session.get(url, timeout=aiohttp.ClientTimeout(total=10)) as r:
        return await r.text()

async def fetch_all(urls):
    async with aiohttp.ClientSession() as session:
        tasks = [fetch(session, url) for url in urls]
        return await asyncio.gather(*tasks, return_exceptions=True)

if __name__ == "__main__":
    urls = ["https://example.com"] * 100
    results = asyncio.run(fetch_all(urls))
    print(f"got {len(results)} responses")
```

**性能对比**：
- 串行 requests：100 × 100ms = 10s
- 多线程（10 worker）：~1s
- asyncio：~0.2s（更省内存）

## 12.9 实战：限流 + 重试

```python
import asyncio
import random

class RateLimiter:
    def __init__(self, max_concurrent):
        self.sem = asyncio.Semaphore(max_concurrent)
    
    async def __aenter__(self):
        await self.sem.acquire()
    
    async def __aexit__(self, *args):
        self.sem.release()

async def fetch_with_retry(url, max_retries=3):
    for attempt in range(max_retries):
        try:
            return await actual_fetch(url)
        except Exception as e:
            if attempt == max_retries - 1:
                raise
            wait = 2 ** attempt + random.random()
            await asyncio.sleep(wait)

async def main():
    limiter = RateLimiter(max_concurrent=10)
    
    async def task(url):
        async with limiter:
            return await fetch_with_retry(url)
    
    urls = [...]
    results = await asyncio.gather(*[task(u) for u in urls])
```

```python
import asyncio

for task in asyncio.all_tasks():
```

### Python 3.11+ 的 TaskGroup（更安全）

```python
async def main():
    async with asyncio.TaskGroup() as tg:
```

## 12.11 asyncio vs 线程：何时选哪个

| | asyncio | 线程 |
|--|---------|------|
| 并发量 | 数千-数万 | 数十-数百 |
| 切换成本 | 极低（用户态） | 中等（OS 调度） |
| 内存占用 | 极低（每协程几 KB） | 高（每线程 ~8MB 栈） |
| 代码风格 | 必须 async/await | 普通 |
| 阻塞库 | 不能用 | 可以用 |

**判断**：
- 新项目 + 高并发 I/O → asyncio
- 调用现有阻塞库 → 线程
- CPU 密集 → 进程

## 12.12 面试常问

### Q1：async/await 和多线程的区别？

**答**：
- async：单线程协作式调度，遇到 await 主动让出
- 线程：操作系统抢占式调度，遇到 I/O 系统切换
- async 资源占用低、并发高，但不能用阻塞库
- 线程上手简单，能直接用现有库，但开销大

### Q2：协程对象、Task、Future 的区别？

**答**：
- 协程对象：调用 `async def` 函数得到的"未启动"对象
- Task：被事件循环调度的协程，是 Future 子类
- Future：底层的"未完成的值"

### Q3：await 和 yield from 的关系？

**答**：在生成器协程时代用 `yield from`，async/await 是新语法。await 限定只能等 awaitable（coroutine/Future/Task），而 yield from 可等任意可迭代对象。

### Q4：以下代码会怎样？

```python
async def main():
    print("a")
    asyncio.create_task(asyncio.sleep(1))
    print("b")

asyncio.run(main())
```

**答**：输出 `a` `b`，且收到 `Task was destroyed but it is pending!` 警告。create_task 后没 await，main 立即返回，task 被丢弃。

### Q5：能否在 sync 代码里运行 async 函数？

**答**：可以，用 `asyncio.run(coro())`。但不能在 async 函数内再用 `asyncio.run`（会报错"already running event loop"），改用 `await`。

### 练习 12.2（中级）

实现 `async_map`：异步地对列表的每个元素应用 async 函数，限制并发为 5：

<details>
<summary>答案</summary>

```python
import asyncio

async def async_map(func, items, concurrency=5):
    sem = asyncio.Semaphore(concurrency)
    
    async def wrap(item):
        async with sem:
            return await func(item)
    
    return await asyncio.gather(*[wrap(it) for it in items])
```
</details>

### 练习 12.4（高级）

实现一个 async 队列消费者，使用 asyncio.Queue：

<details>
<summary>答案</summary>

```python
import asyncio

async def producer(q, n):
    for i in range(n):
        await q.put(i)
    for _ in range(3):
        await q.put(None)    # 哨兵

async def consumer(q, name):
    while True:
        item = await q.get()
        if item is None:
            break
        print(f"[{name}] got {item}")
        await asyncio.sleep(0.01)
        q.task_done()

async def main():
    q = asyncio.Queue(maxsize=10)
    workers = [asyncio.create_task(consumer(q, f"w{i}")) for i in range(3)]
    await producer(q, 30)
    await asyncio.gather(*workers)

asyncio.run(main())
```
</details>

## 本章总结

| 概念 | 一句话 |
|------|--------|
| 协程对象 | `async def f(); f()` 返回，未运行 |
| 事件循环 | 单线程调度器，跑 task 与 I/O |
| await | 暂停协程，让出控制权 |
| Task | 被调度执行的协程 |
| gather | 并发等多个 |
| async with / for | 异步上下文 / 迭代 |
| 不要在 async 中调用阻塞代码 | 用 `run_in_executor` 或异步库 |
