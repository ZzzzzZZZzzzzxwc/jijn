## 1. 模块与包



> **🤔 为什么需要模块系统？**
> 一个文件写几万行代码？不可能维护。模块让你把代码拆成多个文件，各司其职。包让你把相关模块组织成目录结构。这就是"分而治之"在代码组织上的体现。
>
> **🎯 类比理解**
> - 模块 = 一本书的一个章节（独立完整，可单独阅读）
> - 包 = 一整本书（多个章节组织在一起）
> - import = 从书架上取一本书来参考
>
> **💡 设计思路**
> Python 的模块系统遵循**命名空间隔离**原则：每个模块是独立的命名空间，不同模块的同名变量互不干扰。`import` 本质就是"执行那个 .py 文件，把里面的名字放到一个命名空间里给你用"。
>
> **🔄 举一反三**
> - 一个功能对应一个模块（单一职责）
> - 相关模块放同一个包（高内聚）
> - `__init__.py` 控制包的"公开接口"——不是所有东西都该暴露
> - 思考：为什么 Python 搜索模块有优先级？防止你的文件跟标准库重名时出问题

### 导入方式

```python
import os                          # 整个模块
from os.path import join, exists   # 指定内容
import numpy as np                 # 别名

# 条件导入（兼容不同环境）
try:
    import ujson as json  # 性能更好
except ImportError:
    import json           # 标准库兜底
```

### 包结构

```
my_package/
├── __init__.py        # 包初始化（控制导出）
├── module_a.py
├── module_b.py
└── sub_package/
    ├── __init__.py
    └── module_c.py
```

```python
# __init__.py — 控制 from package import * 的行为
from .module_a import ClassA
__all__ = ['ClassA']  # 只暴露这些
```

### `__name__` 与 `__main__`

> **💡 思路：让文件既能被导入也能被直接运行**

```python
def main():
    print("主程序逻辑")

if __name__ == "__main__":
    # 直接运行时执行，被 import 时不执行
    main()
```

> **🔄 举一反三**
> - 这个模式让你写可测试的脚本——导入时只加载函数，不执行副作用
> - 类似 C 的 `int main()` 入口点概念

## 2. 描述符 Descriptor

> **🤔 为什么需要描述符？**
> 当多个类的多个属性需要相同的验证逻辑时（如"必须为正数"、"必须为字符串"），描述符让你把验证逻辑定义一次，到处复用。
>
> **🎯 类比理解**
> 描述符 = 智能表单字段。你定义一个"必须是正整数的字段模板"，然后在多个表单上复用它。每次填写时自动验证。
>
> **💡 设计思路**
> 描述符协议：定义了 `__get__`/`__set__`/`__delete__` 的类。当它作为**类属性**时，Python 会在属性访问时自动调用这些方法。
>
> 本质是**拦截属性访问**——在"读取/设置属性"这个动作上加钩子。property 底层就是描述符。
>
> **🔄 举一反三**
> - ORM 框架的字段类型（Django 的 IntegerField、CharField）就是描述符
> - property 是最简单的描述符
> - 描述符是 Python 属性访问机制的底层基础

```python
class Positive:
    """确保属性值为正数的描述符"""
    def __set_name__(self, owner, name):
        self.name = f"_{name}"
    
    def __get__(self, obj, objtype=None):
        if obj is None: return self
        return getattr(obj, self.name, None)
    
    def __set__(self, obj, value):
        if value <= 0:
            raise ValueError(f"必须为正数, 得到 {value}")
        setattr(obj, self.name, value)

class Product:
    price = Positive()   # 复用！
    quantity = Positive() # 复用！
    
    def __init__(self, price, quantity):
        self.price = price
        self.quantity = quantity

p = Product(10, 5)   # ✅
# Product(-1, 5)     # ❌ ValueError
```

## 3. 文件操作

> **🤔 为什么文件操作很重要？**
> 程序的数据不能只活在内存里——关机就没了。文件是最基本的持久化手段。配置文件、日志、数据导出...都离不开文件操作。
>
> **🎯 类比理解**
> 文件操作 = 图书馆借阅系统。打开(open) = 借书，读写 = 翻阅/写笔记，关闭(close) = 还书。忘记还书(不关文件) = 资源泄漏。
>
> **💡 设计思路**
> 永远用 `with open()` 而不是手动 open/close。理由：
> 1. 自动关闭，不会忘
> 2. 异常时也能正确关闭
> 3. 代码更简洁
>
> **🔄 举一反三**
> - 读大文件 → 逐行迭代 `for line in f`，不用 `readlines()`
> - 路径操作 → 用 pathlib（面向对象），不用 os.path（字符串拼接）
> - 结构化数据 → json/csv/pickle，别自己造格式

### 基本读写

```python
# 写
with open("test.txt", "w", encoding="utf-8") as f:
    f.write("Hello\n")

# 读
with open("test.txt", "r", encoding="utf-8") as f:
    for line in f:  # 逐行，省内存
        print(line.strip())
```

### pathlib（推荐）

> **💡 思路：路径不是字符串，是对象——用对象操作更直观**

```python
from pathlib import Path

p = Path("project/src/main.py")
p.parent       # Path('project/src')
p.name         # 'main.py'
p.suffix       # '.py'
p.exists()     # True/False

# 拼接（用 / 运算符，比 os.path.join 直观）
full = Path("/home") / "user" / "file.txt"

# 遍历
for f in Path(".").glob("**/*.py"):
    print(f)

# 快速读写
Path("f.txt").write_text("hello", encoding="utf-8")
content = Path("f.txt").read_text(encoding="utf-8")
```

### JSON 操作

```python
import json

data = {"name": "小明", "scores": [85, 92]}

# 写入文件
with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

# 读取文件
with open("data.json") as f:
    loaded = json.load(f)

# 字符串互转
s = json.dumps(data, ensure_ascii=False)
d = json.loads(s)
```

## 4. 正则表达式

> **🤔 为什么需要正则表达式？**
> 当你要从文本中"按规则找东西"时：找所有邮箱、提取日期、验证手机号...字符串方法太笨了（只能找固定文本），正则让你描述"模式"。
>
> **🎯 类比理解**
> 正则表达式 = 文本搜索的"通配符加强版"。
> - 字符串 find = 拿着照片找人（只能找完全一样的）
> - 正则 = 拿着画像找人（"戴眼镜的高个子男性" → 模式匹配）
>
> **💡 设计思路**
> 正则是一种**小型语言**（DSL），用特殊符号描述文本模式：
> - `.` = 任意字符
> - `\d` = 数字
> - `+` = 一个或多个
> - `[]` = 字符集
> - `()` = 分组捕获
>
> 学习思路：先记住最常用的几个符号，遇到复杂需求再查。不要试图一次学完。
>
> **🔄 举一反三**
> - 数据清洗：从杂乱文本中提取结构化信息
> - 输入验证：邮箱、手机号、身份证格式检查
> - 日志分析：从日志中提取时间戳、错误码
> - 思考：简单的匹配用字符串方法就够了，正则是"重型武器"——别杀鸡用牛刀

### 核心用法

```python
import re

text = "我的手机 13812345678，邮箱 test@example.com"

# 查找
match = re.search(r'\d{11}', text)
if match:
    print(match.group())  # 13812345678

# 查找所有
phones = re.findall(r'1[3-9]\d{9}', text)

# 替换
result = re.sub(r'\d{11}', '***', text)

# 分割
parts = re.split(r'[,，;；]', "苹果,香蕉，橙子")
```

### 常用模式速记

```python
# 元字符
# .   任意字符    \d  数字    \w  字母数字下划线    \s  空白
# ^   行首        $   行尾    []  字符集           |   或

# 量词
# *   0+次    +   1+次    ?   0或1次
# {n} 恰好n   {n,m} n到m次

# 常用正则（背几个够用）
email = r'[\w.+-]+@[\w-]+\.[\w.]+'
phone = r'1[3-9]\d{9}'
chinese = r'[\u4e00-\u9fa5]+'
date = r'\d{4}-\d{2}-\d{2}'
```

### 分组与命名

> **💡 思路：分组让你不只匹配，还能"提取"出各部分**

```python
pattern = r'(?P<year>\d{4})-(?P<month>\d{2})-(?P<day>\d{2})'
match = re.search(pattern, "日期: 2024-06-15")
if match:
    print(match.group('year'))   # 2024
    print(match.groupdict())     # {'year':'2024', 'month':'06', 'day':'15'}
```

### 性能建议

```python
# 多次使用同一正则 → 预编译
phone_re = re.compile(r'1[3-9]\d{9}')
phone_re.findall(text)  # 比每次 re.findall(pattern, text) 快
```

## 总结：高级特性的思维模型

| 思维原则 | 核心问题 | 对应知识点 |
|---------|---------|-----------|
| 组织 | "代码怎么拆分和管理？" | 模块与包 |
| 惰性 | "能不能用到时再算？" | 生成器、迭代器 |
| 并发 | "能不能同时做多件事？" | asyncio、线程、进程 |
| 拦截 | "能不能在属性访问时加逻辑？" | 描述符 |
| 持久化 | "数据怎么保存和读取？" | 文件操作 |
| 模式匹配 | "怎么从文本里找规律？" | 正则表达式 |

**选择并发方案的决策树：**
```
你的瓶颈是什么？
├── IO等待（网络/磁盘）
│   ├── 大量并发（100+） → asyncio
│   └── 少量并发（<20）  → ThreadPoolExecutor
└── CPU计算（数学/加密）
    └── ProcessPoolExecutor
```

**核心思路：高级特性不是"炫技"，而是解决特定问题的工具。先理解问题本质，再选择合适的工具。**
