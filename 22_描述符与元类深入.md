## 22.0 知识地图

```
属性访问 obj.x 的完整流程（Python 数据模型规范）：
                                                    
1. type(obj).__getattribute__(obj, 'x')  ← 先调用类的 __getattribute__
                                                    
2. ↓ 默认实现按以下顺序查找：
                                                    
   a. type(obj).__mro__ 中查 x
      ├── 是「数据描述符」(__set__或__delete__)：调 __get__ 返回
      └── 否：继续
   
   b. obj.__dict__['x']：找到就返回
   
   c. type(obj).__mro__ 中再查
      └── 是「非数据描述符」(只有__get__)：调 __get__ 返回
   
   d. 都没找到：调 type(obj).__getattr__(obj, 'x')
   
   e. 还没有：AttributeError

数据描述符 > 实例属性 > 非数据描述符
```

## 22.1 描述符协议

### 22.1.1 三种方法

```python
class Descriptor:
    def __get__(self, instance, owner):     # 取值
        ...
    def __set__(self, instance, value):     # 赋值
        ...
    def __delete__(self, instance):         # 删除
        ...
    def __set_name__(self, owner, name):    # 知道自己的名字（3.6+）
        ...
```

### 22.1.2 数据 vs 非数据描述符

| 类型 | 包含 | 优先级 |
|------|------|--------|
| 数据描述符 | `__set__` 或 `__delete__` | **高于**实例属性 |
| 非数据描述符 | 只有 `__get__` | **低于**实例属性 |

### 22.1.3 经典例子：函数本身就是非数据描述符

```python
class Dog:
    def bark(self):       # 函数对象在类层面
        pass

d = Dog()
d.bark    # → bound method（绑定方法）
```

🔬 **机制**：function 类实现了 `__get__`，在通过实例访问时，`__get__` 返回一个 `MethodType`（绑定 self）。

```python
def f(self): pass
print(type(f).__get__)   # <slot wrapper '__get__' of 'function' objects>
```

## 22.2 数据描述符示例：字段类型

```python
class Typed:
    def __init__(self, expected_type):
        self.type = expected_type
    
    def __set_name__(self, owner, name):
        self.private = "_" + name
    
    def __get__(self, instance, owner):
        if instance is None:
            return self                          # 通过类访问返回描述符自身
        return getattr(instance, self.private, None)
    
    def __set__(self, instance, value):
        if not isinstance(value, self.type):
            raise TypeError(f"expected {self.type.__name__}")
        setattr(instance, self.private, value)


class IntField(Typed):
    def __init__(self): super().__init__(int)

class StrField(Typed):
    def __init__(self): super().__init__(str)


class User:
    name = StrField()
    age = IntField()

u = User()
u.name = "Alice"   # ✅
# u.age = "30"     # ❌ TypeError
```

### 22.2.1 ORM 风格的 Field

```python
class Field:
    def __init__(self, *, primary_key=False, nullable=True):
        self.primary_key = primary_key
        self.nullable = nullable
    
    def __set_name__(self, owner, name):
        self.name = name
    
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(self.name)
    
    def __set__(self, instance, value):
        if value is None and not self.nullable:
            raise ValueError(f"{self.name} cannot be null")
        instance.__dict__[self.name] = value

class IntegerField(Field): ...
class CharField(Field): 
    def __init__(self, max_length, **kw):
        super().__init__(**kw)
        self.max_length = max_length

class Model:
    @classmethod
    def fields(cls):
        return {n: v for n, v in vars(cls).items() if isinstance(v, Field)}

class User(Model):
    id = IntegerField(primary_key=True)
    name = CharField(max_length=50)

print(User.fields())   # {'id': IntegerField(...), 'name': CharField(...)}
```

> Django 的 ORM 字段就是这种思路（外加元类自动收集）。

## 22.3 内置描述符

### property = 数据描述符

```python
class C:
    @property
    def x(self):
        return self._x
    
    @x.setter
    def x(self, value):
        self._x = value

# 等价于
class C:
    x = property(fget=..., fset=...)
# property 的 __set__ 让它是数据描述符
```

### staticmethod / classmethod = 非数据描述符

```python
class staticmethod:
    def __init__(self, func): self.func = func
    def __get__(self, instance, owner): return self.func    # 不绑定 self

class classmethod:
    def __init__(self, func): self.func = func
    def __get__(self, instance, owner):
        return MethodType(self.func, owner)   # 绑定 cls
```

### 22.3.4 cached_property（3.8+）

```python
from functools import cached_property

class Data:
    @cached_property
    def expensive(self):
        return slow_compute()
```

🔬 **实现**：第一次访问时计算并写入 `instance.__dict__[name]`。后续访问直接命中实例属性，跳过描述符（因为 cached_property 是**非数据描述符**，实例属性优先级高）。

## 22.4 元类基础

### 22.4.1 类也是对象

```python
class Foo: pass

print(type(Foo))       # <class 'type'>
print(type(type))      # <class 'type'>      type 是自己的元类
```

### 22.4.2 type 创建类的两种方式

```python
# 用法 1：查询类型
type(42)               # <class 'int'>

# 用法 2：动态创建类
Foo = type("Foo", (Base,), {"x": 1, "method": lambda self: None})
```

### 22.4.3 自定义元类

```python
class Meta(type):
    def __new__(mcs, name, bases, namespace, **kwargs):
        print(f"creating class {name}")
        return super().__new__(mcs, name, bases, namespace)
    
    def __init__(cls, name, bases, namespace, **kwargs):
        print(f"initializing class {name}")
        super().__init__(name, bases, namespace)

class MyClass(metaclass=Meta):
    pass
# 输出:
# creating class MyClass
# initializing class MyClass
```

### 22.4.4 元类 vs 类装饰器

很多元类的工作能用类装饰器或 `__init_subclass__` 替代：

```python
# 元类
class Registry(type):
    classes = []
    def __new__(mcs, name, bases, ns):
        cls = super().__new__(mcs, name, bases, ns)
        mcs.classes.append(cls)
        return cls

# 等价的 __init_subclass__（3.6+，更简单）
class Plugin:
    classes = []
    def __init_subclass__(cls, **kw):
        super().__init_subclass__(**kw)
        Plugin.classes.append(cls)

class A(Plugin): pass
class B(Plugin): pass
print(Plugin.classes)   # [A, B]
```

> **💡 推荐**：90% 场景用 `__init_subclass__` 或装饰器。元类只在你**真的需要改变类创建过程**时用。

## 22.5 实战：迷你 ORM

```python
class Field:
    def __set_name__(self, owner, name): self.name = name
    def __get__(self, instance, owner):
        if instance is None: return self
        return instance.__dict__.get(self.name)
    def __set__(self, instance, value):
        instance.__dict__[self.name] = value

class IntegerField(Field): pass
class CharField(Field): pass


class ModelMeta(type):
    def __new__(mcs, name, bases, ns):
        # 收集所有字段
        fields = {k: v for k, v in ns.items() if isinstance(v, Field)}
        ns["_fields"] = fields
        ns.setdefault("_table", name.lower())
        return super().__new__(mcs, name, bases, ns)


class Model(metaclass=ModelMeta):
    def __init__(self, **kwargs):
        for k, v in kwargs.items():
            setattr(self, k, v)
    
    def insert_sql(self):
        cols = ", ".join(self._fields)
        placeholders = ", ".join("?" for _ in self._fields)
        values = tuple(getattr(self, k) for k in self._fields)
        return f"INSERT INTO {self._table} ({cols}) VALUES ({placeholders})", values


class User(Model):
    id = IntegerField()
    name = CharField()


u = User(id=1, name="Alice")
print(u.insert_sql())
# ('INSERT INTO user (id, name) VALUES (?, ?)', (1, 'Alice'))
```

→ Django ORM 的简化雏形。

## 22.6 实战：单例元类

```python
class Singleton(type):
    _instances = {}
    
    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            cls._instances[cls] = super().__call__(*args, **kwargs)
        return cls._instances[cls]


class Database(metaclass=Singleton):
    def __init__(self, url):
        self.url = url

a = Database("...")
b = Database("...")
assert a is b
```

> `__call__` 在元类上 = 拦截"调用类"这个行为，即拦截实例化。

## 22.7 __init_subclass__：元类的轻量替代

```python
class Plugin:
    plugins = {}
    
    def __init_subclass__(cls, *, name=None, **kwargs):
        super().__init_subclass__(**kwargs)
        Plugin.plugins[name or cls.__name__] = cls

class JsonParser(Plugin, name="json"):
    pass

class CsvParser(Plugin, name="csv"):
    pass

print(Plugin.plugins)
# {'json': <class 'JsonParser'>, 'csv': <class 'CsvParser'>}
```

✅ 不需要元类就能实现自动注册。

## 22.8 __set_name__ vs 元类

`__set_name__`（3.6+）让描述符**知道自己的字段名**，不需要元类配合：

```python
class Field:
    def __set_name__(self, owner, name):
        self.name = name           # 自动得到字段名

class User:
    name = Field()                  # __set_name__ 被调用，传入 'name'
```

→ 大幅减少了对元类的依赖。

## 22.9 ABC 与元类

抽象基类用元类实现：

```python
from abc import ABC, ABCMeta, abstractmethod

class Shape(metaclass=ABCMeta):     # 或继承 ABC（更简洁）
    @abstractmethod
    def area(self): ...

# Shape()  # ❌ 不能实例化
```

ABCMeta 还支持"虚拟子类"（不继承也能 `isinstance` 通过）：

```python
class MyList: pass

from collections.abc import Sequence
Sequence.register(MyList)

print(isinstance(MyList(), Sequence))   # True
```

## 22.10 常见误区

### ❌ 误区 1：滥用元类

> "Metaclasses are deeper magic than 99% of users should ever worry about." — Tim Peters

能用 `__init_subclass__` / 装饰器 / 描述符解决就别用元类。

### ❌ 误区 2：描述符放在实例上

```python
class Foo:
    def __init__(self):
        self.field = Field()    # ❌ 描述符必须在「类」上才生效
```

### ❌ 误区 3：忘了 instance is None 的情况

```python
class Field:
    def __get__(self, instance, owner):
        if instance is None:
            return self          # 通过类访问时返回描述符自身
        return instance.__dict__.get(self.name)
```

不处理这个会让 `User.name` 报错。

## 22.11 面试常问

### Q1：属性查找顺序？

**答**：见 22.0 知识地图。**数据描述符 > 实例属性 > 非数据描述符 > __getattr__**。

### Q2：方法为什么能自动绑定 self？

**答**：function 类实现了 `__get__`（非数据描述符），实例访问方法时返回 bound method，自动把 self 绑定。

### Q3：元类的 `__new__` 和 `__init__` 区别？

**答**：
- 元类的 `__new__`：创建类对象
- 元类的 `__init__`：初始化类对象
- 类的 `__new__`：创建实例
- 类的 `__init__`：初始化实例

### Q4：什么时候必须用元类？

**答**：极少。需要在**类创建时**就动作（不是子类创建时）的场景。如 ABCMeta 注册虚拟子类、SQLAlchemy 的复杂映射。

### Q5：__init_subclass__ 与元类区别？

**答**：
- 元类：在**类自身**被创建时执行（包括基类）
- __init_subclass__：在**子类**被创建时执行（基类自身不触发）

## 22.12 练习题

### 练习 22.1（中级）

写一个 `Validated` 描述符基类，子类只需实现 `validate(value)`。

<details><summary>答案</summary>

```python
class Validated:
    def __set_name__(self, owner, name):
        self.private = "_" + name
    def __get__(self, instance, owner):
        if instance is None: return self
        return getattr(instance, self.private, None)
    def __set__(self, instance, value):
        self.validate(value)
        setattr(instance, self.private, value)
    def validate(self, value):
        raise NotImplementedError

class Positive(Validated):
    def validate(self, value):
        if value <= 0:
            raise ValueError("must be positive")
```
</details>

### 练习 22.2（高级）

不用元类，用 `__init_subclass__` 实现"所有子类自动注册到全局列表"。

<details><summary>答案</summary>

```python
class Plugin:
    registry = []
    def __init_subclass__(cls, **kwargs):

        Plugin.registry.append(cls)
```
</details>

### 练习 22.3（专家）

实现一个 `@dataclass` 的简化版本（用元类或 __init_subclass__）。

<details><summary>答案要点</summary>

```python
import inspect

def autoinit(cls):
    annotations = cls.__annotations__
    init_lines = ["def __init__(self, " + ", ".join(annotations) + "):"]
    for name in annotations:
        init_lines.append(f"    self.{name} = {name}")
    exec("\n".join(init_lines), {}, ns := {})
    cls.__init__ = ns["__init__"]
    return cls

@autoinit
class User:
    name: str
    age: int
```
</details>
