## 16.1 ★ 经典陷阱题（写出输出）

### 题 1：默认参数

```python
def f(a, b=[]):
    b.append(a)
    return b

print(f(1))
print(f(2))
print(f(3, []))
print(f(4))
```

<details>
<summary>答案与解释</summary>

```
[1]
[1, 2]
[4]
[1, 2, 4]
```

**解释**：默认参数 `b=[]` 在函数定义时**只创建一次**，存在 `f.__defaults__` 中。每次未传 `b` 时复用同一个列表对象。

📖 **详解**：[§7.6 默认参数陷阱](07_引用与内存机制.md#76--默认参数陷阱)
</details>

### 题 2：闭包

```python
funcs = [lambda: i for i in range(3)]
print([f() for f in funcs])
```

<details>
<summary>答案</summary>

```
[2, 2, 2]
```

**解释**：所有 lambda 共享同一个 cell，cell 中的 `i` 在循环结束后是 2。

修复：`[lambda i=i: i for i in range(3)]`，默认参数在定义时绑定值。

📖 **详解**：[§8.4.4 闭包中的循环陷阱](08_作用域与LEGB深入.md#844--闭包中的循环陷阱)
</details>

### 题 3：is vs ==

```python
a = 256
b = 256
print(a is b)

a = 257
b = 257
print(a is b)

a = "hello"
b = "hello"
print(a is b)

a = "hello world!"
b = "hello world!"
print(a is b)
```

<details>
<summary>答案</summary>

```
True       # 小整数缓存 [-5, 256]
False      # 257 不缓存
True       # 短字符串驻留
True/False # 长字符串驻留与否依赖实现
```

**结论**：判断身份用 `is`，判断值用 `==`。除了 `is None` 外，几乎不用 `is`。

📖 **详解**：[§7.4.2 小整数缓存陷阱](07_引用与内存机制.md#742--小整数缓存陷阱)
</details>

### 题 4：列表乘法

```python
matrix = [[0] * 3] * 3
matrix[0][0] = 1
print(matrix)
```

<details>
<summary>答案</summary>

```
[[1, 0, 0], [1, 0, 0], [1, 0, 0]]
```

**解释**：`[[0]*3]*3` 是把同一个内层列表的引用复制 3 次，3 行是同一对象。

修复：`[[0]*3 for _ in range(3)]`

📖 **详解**：[§7.10 场景 2：避免意外共享](07_引用与内存机制.md#场景-2避免意外共享)
</details>

### 题 5：+=

```python
a = (1, 2)
b = a
b += (3,)
print(a, b, a is b)

a = [1, 2]
b = a
b += [3]
print(a, b, a is b)
```

<details>
<summary>答案</summary>

```
(1, 2) (1, 2, 3) False        # 元组没有 __iadd__，b 重新指向新元组
[1, 2, 3] [1, 2, 3] True      # 列表 __iadd__ 原地修改
```

📖 **详解**：[§7.3.3 += 的双重身份](07_引用与内存机制.md#733--的双重身份)
</details>

### 题 6：UnboundLocalError

```python
x = 10
def f():
    print(x)
    x = 20
f()
```

<details>
<summary>答案</summary>

`UnboundLocalError`。

**解释**：函数编译时扫描赋值，将 `x` 标记为 local。运行时 `print(x)` 用 LOAD_FAST 查找 local x，但 x 还未赋值。

修复：`global x` 或不在函数内重新赋值。

📖 **详解**：[§8.2.3 仅在使用时查找](08_作用域与LEGB深入.md#823--仅在使用时查找不在赋值时)
</details>

### 题 7：类作用域

```python
class A:
    x = 10
    items = [x for _ in range(3)]
print(A.items)
```

<details>
<summary>答案</summary>

`NameError: name 'x' is not defined`。

**解释**：推导式有自己的作用域，看不到类作用域的 x。

修复：
```python
class A:
    x = 10
    items = [x for x in [x] * 3]   # 内层 x 来自外层 x
```

📖 **详解**：[§8.5 类作用域的特殊性](08_作用域与LEGB深入.md#85-类作用域的特殊性)
</details>

### 题 8：浅拷贝

```python
import copy
a = [[1, 2], [3, 4]]
b = a.copy()
b[0].append(99)
b.append([5, 6])
print(a)
print(b)
```

<details>
<summary>答案</summary>

```
[[1, 2, 99], [3, 4]]                ← a 的内层共享，被 b 改了
[[1, 2, 99], [3, 4], [5, 6]]
```

**解释**：浅拷贝复制外层，内层共享。`b.append([5,6])` 只改 b 的外层。

📖 **详解**：[§7.7 拷贝：浅拷贝 vs 深拷贝](07_引用与内存机制.md#77-拷贝浅拷贝-vs-深拷贝)；自定义协议见 [§7.7.4 `__copy__` / `__deepcopy__` 协议](07_引用与内存机制.md#774--__copy__--__deepcopy__-协议)
</details>

### 题 10：生成器表达式作用域

```python
x = 10
gen = (x for _ in range(3))
x = 20
print(list(gen))
```

<details>
<summary>答案</summary>

```

```

**解释**：生成器表达式的**第一个 for 的可迭代对象**在创建时求值，但内层名字在迭代时才查找。这里 x 在迭代时已是 20。

<details>
<summary>答案</summary>

### 题 13：装饰器副作用

```python
def deco(func):
    print(f"装饰 {func.__name__}")
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@deco
def hello():
    print("hi")
```

<details>
<summary>答案</summary>

模块加载时就打印 `装饰 hello`。这说明装饰器**在定义时就执行**。

调用 `hello()` 才打印 `hi`。

📖 **详解**：[§23.1 装饰器的字节码语义](23_装饰器上下文实现与monkey_patch.md#231-装饰器的字节码语义)

### 题 14：异常变量泄漏？

```python
try:
    raise ValueError("oops")
except ValueError as e:
    print(e)

print(e)        # 这行会怎样？
```

<details>
<summary>答案</summary>

`NameError: name 'e' is not defined`（Python 3）。

**解释**：Python 3 中 except 子句结束时**自动 del e**，避免循环引用导致内存泄漏（异常对象引用 traceback，traceback 引用 frame，frame 引用 e）。

📖 **详解**：[§5.3.5 异常链 + except 块自动 del](05_控制流与异常处理篇.md#535--异常链raise-x-from-y)
</details>

### 题 15：协程未 await

```python
import asyncio

async def hello():
    print("hello")

async def main():
    hello()      # 不 await
    print("done")

asyncio.run(main())
```

<details>
<summary>答案</summary>

```
done

```

**解释**：`hello()` 只创建协程对象，不执行。

修复：`await hello()` 或 `asyncio.create_task(hello())`。

📖 **详解**：[§12.2 协程对象：调用 ≠ 执行](12_异步编程深入.md#122-协程对象调用--执行)；[§12.7 陷阱 1：忘记 await](12_异步编程深入.md#1217-常见陷阱)
</details>

## 16.2 概念辨析高频题

### Q1：is 和 == 的区别？

**答**：`is` 比较身份（同一对象），`==` 比较值（调用 `__eq__`）。除了与 None/单例比较外，用 `==`。

### Q2：可变和不可变对象？

**答**：可变（list/dict/set/自定义类）能原地修改；不可变（int/str/tuple/frozenset）任何"修改"都是创建新对象。函数传参时，可变对象的修改会反映到外部。

### Q3：浅拷贝和深拷贝？

**答**：浅拷贝（`.copy()` / `[:]` / `copy.copy`）只复制外层，内层共享引用；深拷贝（`copy.deepcopy`）递归复制所有层。

### Q4：闭包是什么？

**答**：内层函数引用外层函数的局部变量，外层函数返回后，被引用的变量保存在 cell 对象中（`func.__closure__`），形成闭包。

### Q5：装饰器原理？

**答**：装饰器是高阶函数：接收函数返回函数。`@deco` 等价于 `f = deco(f)`。带参装饰器是返回装饰器的工厂。

### Q6：生成器与迭代器？

**答**：迭代器实现 `__iter__` 和 `__next__`；生成器是用 `yield` 写的迭代器，自动维护状态。`yield` 暂停函数，把值返给调用者。

### Q7：GIL 是什么？

**答**：CPython 的全局互斥锁，保证同一时刻只有一个线程执行字节码。存在原因是简化引用计数的线程安全。CPU 密集多线程不快，I/O 密集和 C 扩展能受益。

### Q8：进程、线程、协程区别？

**答**：
- 进程：独立内存，OS 调度，开销最大
- 线程：共享内存，OS 调度，受 GIL 限制
- 协程：用户态调度，单线程内并发，开销最小，需要 async/await

### Q9：Python 是动态类型还是强类型？

**答**：动态 + 强类型。
- 动态：变量无类型，对象有类型，运行时确定
- 强：不会隐式类型转换（`"1" + 1` 报错，不像 JS）

### Q10：`__new__` 和 `__init__`？

**答**：
- `__new__`：创建实例（分配内存），返回实例
- `__init__`：初始化实例（已存在），无返回
- 单例模式重写 `__new__`，普通初始化用 `__init__`

### Q11：Python 多继承的 MRO？

**答**：用 C3 线性化算法。可通过 `Class.__mro__` 查看。`super()` 按 MRO 顺序调用。

### Q12：什么是鸭子类型？

**答**：不关心对象类型，只关心它有什么方法。"长得像鸭子，叫得像鸭子，就是鸭子。"

### Q13：`*args` 和 `**kwargs`？

**答**：
- `*args`：收集位置参数为元组
- `**kwargs`：收集关键字参数为字典
- 调用时 `*args` 解包列表/元组，`**kwargs` 解包字典

### Q14：列表推导和生成器表达式？

**答**：
- 列表推导 `[x for x in ...]`：立即生成全部，占内存
- 生成器表达式 `(x for x in ...)`：惰性，省内存

### Q15：`__slots__` 是什么？

**答**：类属性，限定实例只能有指定属性。优势：节省内存、访问稍快。代价：不能动态加属性，不能多继承（除非父类也有 __slots__）。

### Q16：deepcopy 怎么处理循环引用？

**答**：内部用 memo 字典（id → copy）追踪已复制对象，遇到已复制的直接返回 memo 中的版本。

### Q17：Python 怎么管理内存？

**答**：
1. 主：引用计数（归零立即回收）
2. 辅：分代 GC（清理循环引用）
3. 内存池（pymalloc）减少小对象分配开销

### Q18：with 语句原理？

**答**：实现 `__enter__` 和 `__exit__` 的对象。`__enter__` 返回值赋给 `as` 变量。`__exit__` 在退出时调用（异常或正常）。

### Q19：Python 的 `__main__` 是什么？

**答**：当模块被直接运行时，`__name__ == "__main__"`；被 import 时，`__name__` 是模块名。常用于：

```python
if __name__ == "__main__":
    main()
```

### Q20：global 和 nonlocal 区别？

**答**：
- `global`：声明操作模块级变量，跳过 Enclosing
- `nonlocal`：声明操作最近的外层函数变量（不能是 Global）

## 16.3 综合编程题

### 题 1：实现 LRU 缓存

```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity):
        self.cap = capacity
        self.cache = OrderedDict()
    
    def get(self, key):
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)
        return self.cache[key]
    
    def put(self, key, value):
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.cap:
            self.cache.popitem(last=False)
```

### 题 2：实现 retry 装饰器

```python
from functools import wraps
import time

def retry(times=3, delay=1, exceptions=(Exception,)):
    def deco(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            last_exc = None
            for i in range(times):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    last_exc = e
                    if i < times - 1:
                        time.sleep(delay)
            raise last_exc
        return wrapper
    return deco
```

### 题 8：实现简单的 ORM 字段

```python
class Field:
    def __set_name__(self, owner, name):
        self._name = f"_{name}"
    
    def __get__(self, obj, owner):
        return getattr(obj, self._name, None)
    
    def __set__(self, obj, value):
        self.validate(value)
        setattr(obj, self._name, value)
    
    def validate(self, value):
        pass

class IntField(Field):
    def validate(self, value):
        if not isinstance(value, int):
            raise TypeError(f"expect int, got {type(value).__name__}")

class User:
    age = IntField()

u = User()
u.age = 20         # ✅
# u.age = "20"     # ❌ TypeError
```

## 16.4 系统设计相关

### Q：怎么设计一个高并发的爬虫？

**答**：
1. **并发模型**：asyncio + aiohttp（数千并发）
2. **限流**：Semaphore 控制最大并发；按域名限速
3. **去重**：bloom filter 或 redis set
4. **重试**：指数退避
5. **断点续爬**：状态持久化（数据库或文件）
6. **分布式**：用 Celery + Redis/RabbitMQ 分发任务

### Q：设计一个全局唯一 ID 生成器？

**答**：
- 雪花算法（Snowflake）：时间戳 + 机器 ID + 序列号
- UUID：可全局唯一但长
- Redis INCR：单点瓶颈
- 数据库自增：限制并发

### Q：怎么实现接口幂等？

**答**：
1. 客户端生成 idempotency_key（UUID）
2. 服务端用 key 作 redis 锁/缓存
3. 重复请求查到 key 直接返回旧结果
4. 数据库层面也可以靠唯一索引保护

## 16.5 调试相关

### Q：Python 报错该怎么读？

**答**：从下往上读 traceback。最底下是抛错的位置，往上是调用链。重点看：
1. 错误类型（NameError? KeyError? AttributeError?）
2. 错误信息（具体哪个名字、哪个 key）
3. 文件和行号

### Q：怎么调试不报错但结果不对的代码？

**答**：
1. `print` 配合二分定位
2. `pdb` 断点：`import pdb; pdb.set_trace()`
3. `breakpoint()`（Python 3.7+）
4. IDE 调试器
5. 写测试用例复现

### Q：怎么找内存泄漏？

**答**：
1. `tracemalloc` 拍快照对比
2. `gc.get_objects()` 看对象数量
3. `objgraph` 画对象引用图
4. 找循环引用、全局缓存、未关闭的资源
