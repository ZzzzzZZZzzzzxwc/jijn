## 1. 函数定义与调用

> **🤔 为什么需要函数？**
> 想象你在做饭，如果每次做番茄炒蛋都要从"怎么打鸡蛋"重新想一遍，效率极低。函数就是把一段逻辑"封装成技能"——学会一次，以后直接调用名字就行。
>
> **🎯 类比理解**
> 函数 = 自动售货机。你投入参数（硬币），机器内部执行逻辑（你不用关心），最后吐出返回值（饮料）。
>
> **💡 设计思路**
> Python 设计函数的核心思想是**"一等公民"**——函数本身也是对象，可以赋值给变量、作为参数传递、作为返回值。这个设计让 Python 支持函数式编程范式。
>
> **🔄 举一反三**
> - 你写了一个"发送邮件"的函数，以后任何地方需要发邮件，直接调用即可
> - 同理，任何"重复出现的逻辑"都应该提取成函数
> - 思考：如果函数不能嵌套、不能传递，装饰器和闭包就不可能存在

### 1.1 基本定义

```python
def function_name(parameters):
    """文档字符串 (docstring)"""
    # 函数体
    return value
```

### 1.2 示例

```python
def greet(name):
    """向用户打招呼"""
    return f"你好, {name}!"

result = greet("小明")
print(result)  # 你好, 小明!
```

### 1.3 空函数

```python
def placeholder():
    """占位函数，暂未实现"""
    pass

def todo():
    ...
```

### 1.4 文档字符串规范

```python
def calculate_area(length, width):
    """
    计算矩形面积。

    Args:
        length (float): 矩形的长度
        width (float): 矩形的宽度

    Returns:
        float: 矩形的面积

    Raises:
        ValueError: 当长度或宽度为负数时
    """
    if length < 0 or width < 0:
        raise ValueError("长度和宽度不能为负数")
    return length * width
```

## 2. 参数类型详解

> **🤔 为什么需要这么多参数类型？**
> 不同场景需要不同灵活度。就像餐厅点菜：有"必点菜"（位置参数）、"可改口味"（默认参数）、"加菜随意"（*args）、"备注特殊要求"（**kwargs）。
>
> **🎯 类比理解**
> - 位置参数 = 填表格的必填项，顺序固定
> - 关键字参数 = 填表格时写上字段名，顺序随意
> - 默认参数 = 表格上已经帮你填好的选项，你可以改
> - *args = "其他补充信息，想写多少写多少"
> - **kwargs = "其他键值对信息，自己起名"
>
> **💡 设计思路**
> Python 参数系统的设计哲学：**从严格到灵活逐级递进**。位置参数保证核心信息不丢，默认参数减少重复，*args/**kwargs 实现"开放扩展"。这就是开闭原则在参数设计中的体现。
>
> **🔄 举一反三**
> - 设计 API 时：核心参数用位置参数，可选配置用默认参数/kwargs
> - Flask/Django 的路由函数大量使用这种模式
> - 思考：为什么 `print()` 能接收任意数量参数？因为它用了 *args

### 2.1 位置参数

```python
def add(a, b):
    return a + b

add(3, 5)  # a=3, b=5
```

### 2.2 关键字参数

```python
def user_info(name, age, city):
    return f"{name}, {age}岁, 来自{city}"

user_info(age=25, name="小红", city="北京")
```

### 2.3 默认参数

```python
def power(base, exp=2):
    return base ** exp

power(3)      # 9
power(2, 10)  # 1024
```

> ⚠️ **陷阱与思路：为什么不能用可变对象做默认值？**
> 因为默认参数在函数**定义时**只创建一次，之后每次调用共享同一个对象。
> 类比：这就像全班同学共用一块白板当"默认草稿纸"，一个人写了东西，下个人还能看到。

```python
# ❌ 错误
def append_to(element, target=[]):
    target.append(element)
    return target

# ✅ 正确 — 每次调用创建新对象
def append_to(element, target=None):
    if target is None:
        target = []
    target.append(element)
    return target
```

### 2.4 *args 和 **kwargs

```python
def calculate_sum(*numbers):
    return sum(numbers)  # numbers 是元组

def print_info(**kwargs):
    for key, value in kwargs.items():
        print(f"  {key}: {value}")  # kwargs 是字典
```

### 2.5 参数组合顺序

```python
# 记忆口诀：位置 → *args → 关键字 → **kwargs
def complex_func(a, b, *args, c=10, **kwargs):
    pass
```

### 2.6 仅限位置 / 仅限关键字

```python
# / 之前：只能位置传递
def divide(a, b, /):
    return a / b

# * 之后：只能关键字传递
def connect(host, port, *, timeout=30):
    pass
```

> **💡 为什么要限制传参方式？**
> 为了 API 的稳定性。如果参数名可能变化，用仅限位置就不会影响调用者。如果参数含义不明显，强制用关键字让代码更可读。

### 2.7 参数解包

```python
numbers = [1, 2, 3]
add(*numbers)  # 等同于 add(1, 2, 3)

config = {"host": "localhost", "port": 8080}
connect(**config)
```

## 3. 返回值

> **🤔 为什么需要返回值？**
> 函数不只是"做事"，更重要的是"给结果"。没有返回值的函数像一个只会做菜但不端上桌的厨师。
>
> **🎯 类比理解**
> return = 快递签收。函数内部处理完（加工），通过 return 把结果"寄回"给调用者。
>
> **💡 设计思路**
> Python 允许返回多个值（本质是返回元组），这比 C/Java 只能返回单一值灵活得多。设计函数时思考：调用者需要什么信息？只给必要的。

```python
def min_max(numbers):
    return min(numbers), max(numbers)  # 返回元组

minimum, maximum = min_max([3, 1, 4, 1, 5, 9])
```

> **🔄 举一反三**
> - 返回字典：适合结果有多个命名字段时
> - 返回 None：表示"没找到"/"操作无结果"
> - 卫语句（提前 return）：减少嵌套，让代码更清晰

```python
# 卫语句思路：先排除异常情况，再处理正常逻辑
def divide(a, b):
    if b == 0:
        return None  # 提前返回
    return a / b
```

## 4. 作用域与命名空间

> **🤔 为什么需要作用域？**
> 如果所有变量都是全局的，一个大项目里变量名冲突会变成噩梦。作用域让变量有"活动范围"，互不干扰。
>
> **🎯 类比理解**
> 作用域 = 房间。同一栋楼里每个房间可以有自己的"小明"，互不冲突。函数里的变量就像房间里的物品，出了门就看不到。
>
> **💡 设计思路（LEGB 规则）**
> Python 查找变量的顺序：Local(本地) → Enclosing(外层函数) → Global(全局) → Built-in(内置)
> 这是一种**就近原则**——先在最近的地方找，找不到再往外扩大范围。

```python
x = "全局"  # Global

def outer():
    x = "外层"  # Enclosing
    def inner():
        x = "内层"  # Local
        print(x)  # 找到 Local 的 x
    inner()

outer()  # 输出: 内层
```

### global 和 nonlocal

```python
counter = 0
def increment():
    global counter  # 声明：我要修改全局的 counter
    counter += 1

def outer():
    count = 0
    def inner():
        nonlocal count  # 声明：我要修改外层函数的 count
        count += 1
    inner()
```

> **🔄 举一反三**
> - 尽量避免 global：全局变量是"公共厕所"，谁都能改，难以追踪 bug
> - nonlocal 是闭包的基础：没有它，闭包就无法修改外层状态
> - 思考：JavaScript 的 var/let/const 作用域规则和 Python 有什么异同？

## 5. 闭包 (Closure)

> **🤔 为什么需要闭包？**
> 有时你需要一个函数"记住"一些私有状态，但又不想用类那么重。闭包就是轻量级的"有记忆的函数"。
>
> **🎯 类比理解**
> 闭包 = 背着书包的学生。函数是学生，闭包变量是书包里的东西。即使离开了教室（外层函数执行完了），书包里的东西还带在身上。
>
> **💡 设计思路**
> 闭包的本质：**内层函数 + 它引用的外层变量**。Python 通过 `__closure__` 属性把外层变量"打包"到内层函数上。这是一种不用类就能实现"状态封装"的方式。
>
> **🔄 举一反三**
> - 装饰器的核心机制就是闭包
> - React 的 Hooks (useState) 概念类似闭包
> - 工厂函数模式：make_multiplier(2) 返回一个"永远乘2"的函数

```python
def make_multiplier(factor):
    def multiply(x):
        return x * factor  # factor 被闭包捕获
    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)
print(double(5))   # 10
print(triple(5))   # 15
```

### 闭包常见陷阱

> **💡 思路：为什么循环里的闭包会出问题？**
> 因为闭包捕获的是**变量的引用**，不是当时的值。循环结束后，变量停留在最后一个值。

```python
# ❌ 所有函数都返回 4
functions = []
for i in range(5):
    def f():
        return i  # 捕获的是 i 的引用，不是值
    functions.append(f)

# ✅ 用默认参数"快照"当前值
functions = []
for i in range(5):
    def f(x=i):  # x=i 在定义时求值，固定住了
        return x
    functions.append(f)
```

## 6. 递归函数

> **🤔 为什么需要递归？**
> 有些问题天然是"自我重复"的结构：文件夹里套文件夹、树的分支上还有分支。递归让你用"我自己"来解决"我自己的子问题"。
>
> **🎯 类比理解**
> 递归 = 俄罗斯套娃。打开一个娃娃，里面还有一个更小的，直到最小的那个（基准条件）为止。
> 也像照镜子：两面镜子对着放，画面无限嵌套。
>
> **💡 设计思路**
> 递归三要素：
> 1. **基准条件**（什么时候停？）— 没有它就无限循环
> 2. **递归调用**（怎么缩小问题？）— 每次调用必须更接近基准条件
> 3. **问题分解**（大问题怎么拆成小问题？）
>
> **🔄 举一反三**
> - 阶乘：n! = n × (n-1)! → 大问题变小问题
> - 文件遍历：遇到文件夹就"递归进入"
> - 分治算法（快排、归并排序）都是递归思想
> - 思考：递归都能改成循环，什么时候该用哪个？

```python
def factorial(n):
    if n <= 1:      # 基准条件（套娃到头了）
        return 1
    return n * factorial(n - 1)  # 递归：问题缩小
```

```python
# 带缓存优化 — 避免重复计算
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
```

> **💡 递归深度限制**：Python 默认限制 1000 层。遇到深递归时，考虑改成循环或增大限制。

## 7. 高阶函数

> **🤔 为什么需要高阶函数？**
> 因为很多操作的"骨架"是一样的，只有"具体做什么"不同。高阶函数让你把"做什么"作为参数传进去，实现代码复用。
>
> **🎯 类比理解**
> 高阶函数 = 万能遥控器。你给它装上不同的"频道按钮"（函数参数），它就能控制不同的设备。
> 或者像外卖平台：平台负责配送（骨架），你决定点什么菜（具体函数）。
>
> **💡 设计思路**
> 高阶函数体现了**"策略模式"**的思想：把变化的部分（策略/算法）抽出来作为参数，不变的部分（流程/骨架）固定。这是"开闭原则"的具体体现。
>
> **🔄 举一反三**
> - `sorted(data, key=func)` — 排序骨架不变，排序依据可换
> - JavaScript 的 `Array.map/filter/reduce` 同理
> - 回调函数、事件处理器都是高阶函数的应用
> - 思考：如果没有高阶函数，你需要为"按名字排""按分数排""按日期排"各写一个排序函数

```python
# map: 对每个元素做同样的事
squares = list(map(lambda x: x**2, [1, 2, 3, 4, 5]))

# filter: 按条件筛选
evens = list(filter(lambda x: x % 2 == 0, range(20)))

# sorted + key: 自定义排序规则
students = [{"name": "小明", "score": 85}, {"name": "小红", "score": 92}]
sorted(students, key=lambda s: s["score"], reverse=True)
```

```python
# 函数组合：把多个小函数串成管道
def compose(*functions):
    def composed(x):
        for f in reversed(functions):
            x = f(x)
        return x
    return composed

transform = compose(lambda x: x**2, lambda x: x*2, lambda x: x+1)
print(transform(3))  # (3+1)*2 = 8, 8**2 = 64
```

## 8. Lambda 表达式

> **🤔 为什么需要 Lambda？**
> 有时你只需要一个"用完即扔"的小函数，专门写 def 取名字太浪费。Lambda 就是"匿名的一次性函数"。
>
> **🎯 类比理解**
> Lambda = 便利贴。def 是正式文件（有名字、存档），lambda 是临时便利贴（写完用完就丢）。
>
> **💡 设计思路**
> Lambda 只能写一个表达式，这是**故意的限制**。Python 之父认为：如果逻辑复杂到需要多行，就应该用 def 给它个名字，让代码更可读。Lambda 的定位就是"简单到不需要名字"。
>
> **🔄 举一反三**
> - 最常用场景：sorted 的 key、map/filter 的参数
> - 如果 lambda 里逻辑超过一行，果断改用 def
> - 其他语言的箭头函数 (JS: `x => x*2`) 就是类似概念

```python
# 一行搞定简单逻辑
square = lambda x: x ** 2
pairs = [(1, 'b'), (2, 'a')]
pairs.sort(key=lambda p: p[1])
```

## 9. 内置函数精选

> **🤔 为什么要知道内置函数？**
> 内置函数是 Python 已经帮你写好的"工具箱"。自己造轮子不仅慢，还容易出 bug。用好内置函数 = 站在巨人肩膀上。
>
> **💡 设计思路**
> Python 的哲学："Batteries included"（自带电池）。常用操作都有现成函数，减少重复劳动。

```python
# 数学: abs, round, pow, max, min, sum, divmod
# 类型: int, float, str, bool, list, tuple, dict, set
# 迭代: enumerate, zip, reversed, all, any, map, filter
# 对象: type, isinstance, id, dir, hasattr, getattr, callable
```

> **🔄 举一反三**
> - 遍历时需要索引？别用 `range(len(x))`，用 `enumerate`
> - 需要同时遍历两个列表？用 `zip`
> - 判断"列表里是否全部/存在满足条件的"？用 `all/any`

## 10. 函数注解与类型提示

> **🤔 为什么需要类型提示？**
> Python 是动态类型语言——灵活但容易出错。类型提示就像给代码加"路标"：不强制，但让人（和工具）更容易理解。
>
> **🎯 类比理解**
> 类型提示 = 快递包裹上的标签。不贴标签也能寄，但贴了标签分拣更快、出错更少。
>
> **💡 设计思路**
> Python 选择了"渐进式类型"——类型提示是可选的，运行时不强制检查。这样既保留了动态语言的灵活性，又能通过 mypy 等工具在开发阶段发现错误。两全其美。
>
> **🔄 举一反三**
> - TypeScript 对 JavaScript 做了同样的事
> - 大型项目必须加类型提示，否则维护成本爆炸
> - IDE 的自动补全依赖类型提示

```python
def greet(name: str) -> str:
    return f"Hello, {name}!"

def find_user(user_id: int) -> str | None:  # Python 3.10+
    users = {1: "小明"}
    return users.get(user_id)
```

## 11. 函数式编程工具

> **🤔 为什么需要函数式编程工具？**
> 函数式编程的核心思想：数据像水一样"流过"一个个函数，每个函数做一步变换。这种模式天然适合数据处理管道。
>
> **🎯 类比理解**
> - `partial` = 把万能遥控器预设成"只控制电视"的专用遥控器
> - `lru_cache` = 把计算结果写在小抄上，下次直接看小抄
> - `reduce` = 滚雪球：从一个小球开始，每次合并一个新元素
>
> **💡 设计思路**
> functools/itertools 的设计理念：提供**组合的积木**，让你用小函数搭建大逻辑。每个工具都是纯的、可组合的，这就是 Unix 哲学 "Do one thing well"。

```python
from functools import partial, lru_cache

# partial: 固定部分参数，生成新函数
square = partial(pow, exp=2)  # pow(base, exp=2)

# lru_cache: 自动缓存结果
@lru_cache(maxsize=128)
def expensive_compute(n):
    return n ** 2
```

> **🔄 举一反三**
> - partial 在 GUI 编程中很常见：`button.bindcommand=partial(handler, arg)`
> - lru_cache 对递归优化效果惊人（斐波那契从 O(2^n) 降到 O(n)）
> - itertools 是处理大数据的利器：chain, groupby, combinations 等

## 总结：函数的思维模型

| 思维原则 | 核心问题 | 对应知识点 |
|---------|---------|-----------|
| 抽象 | "这段逻辑能复用吗？" | 函数定义 |
| 封装 | "调用者需要知道内部细节吗？" | 参数、返回值 |
| 组合 | "能把小函数拼成大功能吗？" | 高阶函数、compose |
| 记忆 | "需要函数记住状态吗？" | 闭包、缓存 |

### 练习 1（基础）

写一个函数 `safe_div(a, b)`：除法，b 为 0 时返回 None。

<details><summary>答案</summary>

```python
def safe_div(a: float, b: float) -> float | None:
    if b == 0:
        return None
    return a / b
```
</details>

### 练习 2（参数）

实现一个函数，接收任意位置参数和关键字参数，返回字典：`{"args": [...], "kwargs": {...}}`。

<details><summary>答案</summary>

```python
def collect(*args, **kwargs):
    return {"args": list(args), "kwargs": kwargs}

print(collect(1, 2, name="Alice"))
# {'args': [1, 2], 'kwargs': {'name': 'Alice'}}
```

### 练习 4（闭包）

实现 `make_adder(n)`，返回一个加 n 的函数：

```python
add5 = make_adder(5)
add5(10)   # 15
```

<details><summary>答案</summary>

```python
def make_adder(n):
    def adder(x):
        return x + n
    return adder
```
</details>

### 练习 5（高阶函数）

实现 `compose(f, g)`，返回 `lambda x: f(g(x))`，并支持任意多个：

```python
double = lambda x: x * 2
inc = lambda x: x + 1
pipe = compose(double, inc, double)
pipe(3)   # 13: (3*2+1)*2 = 14, 实际看实现顺序
```

<details><summary>答案</summary>

```python
from functools import reduce

def compose(*funcs):
    return reduce(lambda f, g: lambda x: f(g(x)), funcs)

# 测试
pipe = compose(lambda x: x*2, lambda x: x+1)
print(pipe(3))   # 8: 先 +1 再 *2
```
</details>
