## 🎓 学习方法

### 1. 手动追踪变量

读到代码不要直接看输出，先在纸上画：
- 内存中有哪些对象
- 每个名字指向哪里
- 引用计数如何变化

### 2. 用 dis 模块看字节码

不确定 Python 在干什么时：

```python
import dis
dis.dis(my_function)
```

这是终极工具。

### 3. 用 id() 验证引用

```python
a = [1, 2, 3]
b = a
print(id(a) == id(b))   # True，同一对象
```

### 4. 每章做练习题

**不动手做，等于没看**。

### 5. 看到陌生概念立刻打开 16 章

[16 章](16_面试陷阱与高频考点.md) 集中了 20+ 高频陷阱，扫一遍就能查漏补缺。

## 📌 规范 vs 实现

**Python 语言规范** (PEP / 官方文档):
- 语法、对象模型、特殊方法协议、import 语义、async/await...

**CPython 实现细节** (用 🔬 标注):
- GIL、引用计数、分代 GC、小整数缓存、字符串驻留、字节码指令、PyObject 内存布局...

→ PyPy / Jython / IronPython 可能行为不同，写跨实现兼容代码时要注意。

## 🎁 给读者的话

Python 看起来简单，但简单语法下隐藏复杂运行时机制：
- 一个普通的 `a = b` 到底做了什么？
- 为什么 `def f(x=[]):` 是个陷阱？
- 为什么多线程跑不快？
- 为什么 `await` 不阻塞？

这些问题，速查文档不会告诉你。这份文档会。

如果你只想"能写出来"，看任何速成教程都行。  
如果你想"知道自己在写什么"，请认真读完，并完成所有练习。
