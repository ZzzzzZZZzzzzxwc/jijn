直接读 **07 → 08 → 10 → 12 → 16**。这五章覆盖 90% 的"为什么"问题。

## 学习方法

1. **先读知识地图**，建立全局观
2. **看到代码先预测输出**，再运行验证
3. **用 `dis` 和 `id` 看底层**，不确定就实证
4. **每章做完练习题**，不动手等于没看
5. **和真实项目结合**，理论 + 工程缺一不可
