## 7.1 一切皆对象

### 7.1.1 概念

Python 中**所有数据都是对象**：整数、字符串、列表、函数、类、模块——全都是。

每个对象都有三个属性：

| 属性 | 含义 | 怎么查看 |
|------|------|----------|
| **身份 (identity)** | 内存中的唯一标识 | `id(obj)` |
| **类型 (type)** | 决定有哪些方法和操作 | `type(obj)` |
| **值 (value)** | 存储的数据 | 直接查看 |

```python
# ─────────────────────────────────────
# 一切皆对象：连数字都是
# ─────────────────────────────────────
x = 42
print(id(x))       # 输出: 140234567890032 （示例，每次运行不同）
print(type(x))     # 输出: <class 'int'>
print(x.bit_length())  # 输出: 6   ← 整数也是对象，有方法

# 函数也是对象
def hello(): pass
print(type(hello))     # 输出: <class 'function'>
hello.custom = "yes"   # 可以给函数加属性！
print(hello.custom)    # 输出: yes
```

### 7.1.2 💡 为什么这样设计

C/Java 中变量是"装东西的盒子"：
```c
int x = 42;   // 变量 x 在栈上分配 4 字节，里面存着 42
```

Python 中变量是"贴在对象上的标签"：
```python
x = 42        # 内存中创建 int 对象 42，名字 x 贴在它上面
```

这个设计的好处：
- **统一性**：所有数据按同样规则处理（GC、传参、赋值）
- **灵活性**：变量可以指向任何类型，不需要预先声明
- **反射友好**：能在运行时检查、修改任何对象

代价：
- **性能开销**：每个对象都有元信息（type 指针、引用计数）
- **小整数都要分配对象**（CPython 用缓存优化了这一点，见 7.4）

## 7.2 名字与对象的关系

### 7.2.1 ★ 核心模型：名字 → 对象

```
                              内存中的对象
                              ┌──────────┐
        ┌──────┐              │ int 42   │
        │  x   │ ───────────► │ id=0x100 │
        └──────┘              └──────────┘
        变量名（标签）         真正的对象
```

**赋值不是复制对象，只是让名字指向对象。**

```python
# ─────────────────────────────────────
# 赋值的真相
# ─────────────────────────────────────
a = [1, 2, 3]    # 创建列表对象，a 指向它
b = a            # b 也指向同一个列表（没有复制！）

print(id(a))     # 输出: 140234567 (示例)
print(id(b))     # 输出: 140234567 ← 同一个 id
print(a is b)    # 输出: True       ← 同一对象

b.append(4)      # 通过 b 修改列表
print(a)         # 输出: [1, 2, 3, 4]  ← a 看到的也变了！
```

**内存示意图**：

```
赋值前后：

a = [1, 2, 3]
        ┌─────────────┐
   a ──►│ [1, 2, 3]   │
        └─────────────┘

b = a
        ┌─────────────┐
   a ──►│ [1, 2, 3]   │◄── b      ← 两个名字指向同一对象
        └─────────────┘

b.append(4)
        ┌─────────────┐
   a ──►│[1, 2, 3, 4] │◄── b      ← 修改的是对象本身
        └─────────────┘
```

### 7.2.2 重新赋值 vs 修改对象

这是初学者最容易混淆的地方：

```python
# ─────────────────────────────────────
# 情形 A：重新赋值（让名字指向新对象）
# ─────────────────────────────────────
a = [1, 2, 3]
b = a
b = [99]          # b 现在指向新列表 [99]，a 不受影响

print(a)          # 输出: [1, 2, 3]
print(b)          # 输出: [99]


# ─────────────────────────────────────
# 情形 B：修改对象（同一对象被改）
# ─────────────────────────────────────
a = [1, 2, 3]
b = a
b.append(99)      # 修改 b 指向的对象（也是 a 指向的对象）

print(a)          # 输出: [1, 2, 3, 99]
print(b)          # 输出: [1, 2, 3, 99]
```

**关键判断标准**：
- `=` 让名字**重新绑定**到新对象
- `.method()` / `[i] = ` / `+=`（对可变对象）**修改原对象**

## 7.3 可变对象 vs 不可变对象

### 7.3.1 分类

| 不可变 (immutable) | 可变 (mutable) |
|--------------------|----------------|
| `int`, `float`, `bool` | `list` |
| `str`, `bytes` | `dict` |
| `tuple`, `frozenset` | `set` |
| `None` | 自定义类（默认） |

**判定方法**：能否通过对象自己的方法修改其内容。
- `lst.append(1)` ✅ 修改了 lst → 可变
- `s.upper()` 返回新字符串，s 没变 → 不可变

### 7.3.2 不可变对象的"修改"是假象

```python
# ─────────────────────────────────────
# 字符串"修改"实际是创建新对象
# ─────────────────────────────────────
s = "hello"
print(id(s))      # 输出: 140234001 (示例)

s = s + " world"  # 创建新字符串，s 重新指向它
print(id(s))      # 输出: 140234999 ← 不同的 id
print(s)          # 输出: hello world
```

```python
# ─────────────────────────────────────
# 整数"修改"也是
# ─────────────────────────────────────
n = 100
print(id(n))      # 输出: 140234500

n += 1            # 等价于 n = n + 1，创建新 int 对象
print(id(n))      # 输出: 140234600 ← 不同的 id（新对象）
```

### 7.3.3 ⚠️ `+=` 的双重身份

```python
# ─────────────────────────────────────
# 同样是 +=，行为完全不同
# ─────────────────────────────────────

# 不可变对象：等价于 a = a + b（重新绑定）
a = [1, 2]
b = a
b += [3]          # 调用 list.__iadd__，原地修改！
print(a)          # 输出: [1, 2, 3]   ← a 也变了

c = (1, 2)
d = c
d += (3,)         # 元组没有 __iadd__，等价于 d = d + (3,)
print(c)          # 输出: (1, 2)      ← c 不变（创建了新元组）
print(d)          # 输出: (1, 2, 3)
```

🔬 **底层原理**：
- `list` 实现了 `__iadd__`，`+=` 调用它做原地修改
- `tuple` 没有实现 `__iadd__`，`+=` 退化为 `__add__` + 重新赋值

```python
# 验证
print([].__iadd__)      # 输出: <method-wrapper '__iadd__' of list object>
print((,).__iadd__)     # 语法错误，但元组确实没有 __iadd__
```

## 7.4 身份比较：is vs ==

### 7.4.1 概念区分

| 操作 | 比较什么 | 调用 |
|------|----------|------|
| `a == b` | **值**是否相等 | `a.__eq__(b)` |
| `a is b` | **身份**是否相同（同一对象） | 比较 `id(a) == id(b)` |

```python
a = [1, 2, 3]
b = [1, 2, 3]
print(a == b)    # 输出: True   ← 值相等
print(a is b)    # 输出: False  ← 不是同一对象

c = a
print(a is c)    # 输出: True   ← 同一对象
```

### 7.4.2 ⚠️ 小整数缓存陷阱

```python
# ─────────────────────────────────────
# 小整数：身份可能"凑巧"相同
# ─────────────────────────────────────
a = 256
b = 256
print(a is b)    # 输出: True   ← CPython 缓存了 -5 到 256

a = 257
b = 257
print(a is b)    # 输出: False  ← 256 之外不缓存

# 但在同一行（同一编译单元）会被优化
a = 1000; b = 1000
print(a is b)    # 输出: True   ← 编译期常量折叠

# 在不同语句
a = 1000
b = 1000
print(a is b)    # 输出: 可能是 True 或 False，取决于实现
```

🔬 **CPython 实现细节**：
- `[-5, 256]` 范围的整数在解释器启动时就预创建（`small_ints` 数组）
- 短字符串（标识符样式）也会被驻留（intern）
- 这是 **CPython 优化**，不是语言规范！其他实现（PyPy）可能不同

### 7.4.3 ★ 黄金法则

```python
# ✅ 与 None 比较：永远用 is
if x is None:    # ✅ 推荐
    pass

if x == None:    # ❌ 不推荐（虽然能工作）
    pass
```

**为什么**：
1. `is` 更快（只比指针）
2. 自定义类可以重载 `__eq__`，可能让 `x == None` 行为诡异
3. None 是单例，全局只有一个，用 `is` 语义更准确

## 7.5 函数传参的真相

### 7.5.1 ★ Python 的传参机制：传对象引用

很多教材说"Python 是按值传递"或"按引用传递"，**两个说法都不准确**。

正确说法：**传递对象的引用**（pass by object reference / pass by sharing）。

```python
# ─────────────────────────────────────
# 函数能否修改外部变量？看对象是否可变
# ─────────────────────────────────────
def modify(x):
    x.append(99)      # 通过引用修改对象本身

def reassign(x):
    x = [99]          # 重新让本地名字 x 指向新对象（外部不受影响）

lst = [1, 2, 3]
modify(lst)
print(lst)            # 输出: [1, 2, 3, 99]   ← 被修改了

lst = [1, 2, 3]
reassign(lst)
print(lst)            # 输出: [1, 2, 3]       ← 没被修改
```

**执行流程图**：

```
modify(lst) 调用：

         调用前                 函数内 x.append(99) 后
    ┌─────────────┐         ┌──────────────────┐
lst─►│ [1, 2, 3]  │      lst─►│ [1, 2, 3, 99]   │
    └─────────────┘         └──────────────────┘
                              ▲
                              │
                              x（函数参数）

→ x 和 lst 指向同一对象
→ 修改对象本身，lst 也"看到"变化


reassign(lst) 调用：

         调用前                 函数内 x = [99] 后
    ┌─────────────┐         ┌─────────────┐
lst─►│ [1, 2, 3]  │      lst─►│ [1, 2, 3]   │
    └─────────────┘         └─────────────┘
                              
                              ┌─────────────┐
                          x ─►│ [99]        │   ← 函数返回后销毁
                              └─────────────┘

→ x 在函数内重新绑定到新对象
→ lst 始终指向 [1,2,3]
```

### 7.5.2 想要"真正的值传递"？显式拷贝

```python
def safe_modify(x):
    x = x.copy()      # 显式拷贝
    x.append(99)
    return x

lst = [1, 2, 3]
new_lst = safe_modify(lst)
print(lst)        # 输出: [1, 2, 3]      ← 原列表不变
print(new_lst)    # 输出: [1, 2, 3, 99]
```

## 7.6 ★★★ 默认参数陷阱

这是 Python 面试最高频陷阱之一。

### 7.6.1 错误示例

```python
# ❌ 看起来正常，实际是 bug
def append_item(item, target=[]):
    target.append(item)
    return target

print(append_item(1))    # 输出: [1]
print(append_item(2))    # 输出: [1, 2]   ← 怎么记住了上次的？！
print(append_item(3))    # 输出: [1, 2, 3]
```

### 7.6.2 🔬 底层原理

**默认参数在函数定义时只创建一次**，不是每次调用时创建。

```python
def append_item(item, target=[]):
    ...

# 等价的内部模型：
def append_item(item, target=_DEFAULT_LIST_OBJECT):
    ...

# _DEFAULT_LIST_OBJECT 在函数定义时创建，存在 append_item.__defaults__
print(append_item.__defaults__)   # 输出: ([],)  这是函数对象的属性

append_item(1)                     # target 指向 _DEFAULT_LIST_OBJECT
                                   # _DEFAULT_LIST_OBJECT 现在是 [1]
print(append_item.__defaults__)   # 输出: ([1],) ← 默认值被改了！
```

### 7.6.3 ✅ 正确写法

```python
# ✅ 用 None 作哨兵值
def append_item(item, target=None):
    if target is None:
        target = []        # 每次调用创建新列表
    target.append(item)
    return target

print(append_item(1))    # 输出: [1]
print(append_item(2))    # 输出: [2]
print(append_item(3))    # 输出: [3]
```

### 7.6.4 💡 什么时候可以利用这个特性？

**故意**用默认参数缓存：

```python
def fib(n, _cache={0: 0, 1: 1}):
    """利用默认参数实现缓存（不推荐，但能工作）"""
    if n not in _cache:
        _cache[n] = fib(n-1) + fib(n-2)
    return _cache[n]

print(fib(50))   # 输出: 12586269025  ← 秒出
```

但更推荐 `@functools.lru_cache`，意图清晰。

## 7.7 拷贝：浅拷贝 vs 深拷贝

### 7.7.1 三种"拷贝"

```python
import copy

a = [[1, 2], [3, 4]]

b = a               # ① 引用赋值（不是拷贝）
c = a.copy()        # ② 浅拷贝（也可以 list(a) 或 a[:]）
d = copy.deepcopy(a) # ③ 深拷贝
```

**内存示意**：

```
a = [[1, 2], [3, 4]]

         ┌──────────┐    ┌──────┐
    a───►│ outer    │───►│[1,2] │
         │          │    └──────┘
         │          │    ┌──────┐
         │          │───►│[3,4] │
         └──────────┘    └──────┘

b = a:
    a, b 都指向 outer

c = a.copy():
         ┌──────────┐    ┌──────┐
    a───►│ outer A  │───►│[1,2] │  ← 注意：内层共享！
         └──────────┘    └──────┘
                          ▲
         ┌──────────┐     │
    c───►│ outer B  │─────┤
         │          │    ┌──────┐
         │          │───►│[3,4] │  ← 内层共享
         └──────────┘    └──────┘

d = deepcopy(a):
         ┌──────────┐    ┌──────┐
    a───►│ outer A  │───►│[1,2] │
         └──────────┘    └──────┘

         ┌──────────┐    ┌──────┐
    d───►│ outer C  │───►│[1,2] │  ← 完全独立
         │          │    └──────┘
         │          │    ┌──────┐
         │          │───►│[3,4] │
         └──────────┘    └──────┘
```

### 7.7.2 行为差异

```python
import copy

a = [[1, 2], [3, 4]]
b = a
c = a.copy()
d = copy.deepcopy(a)

a[0].append(99)        # 修改内层列表

print(b)   # 输出: [[1, 2, 99], [3, 4]]   ← b 是 a，必然变
print(c)   # 输出: [[1, 2, 99], [3, 4]]   ← 浅拷贝共享内层！
print(d)   # 输出: [[1, 2], [3, 4]]       ← 深拷贝完全独立

a.append([5, 6])       # 修改外层列表

print(b)   # 输出: [[1, 2, 99], [3, 4], [5, 6]]   ← 同一对象
print(c)   # 输出: [[1, 2, 99], [3, 4]]            ← 外层独立
print(d)   # 输出: [[1, 2], [3, 4]]
```

### 7.7.3 什么时候用什么

| 需求 | 选择 |
|------|------|
| 只想再起一个名字 | 直接赋值 `b = a` |
| 修改外层不影响原对象 | 浅拷贝（且内层是不可变对象时） |
| 修改任何层都不影响原对象 | 深拷贝 |
| 不可变对象 | 都不需要拷贝 |

## 7.8 引用计数与垃圾回收

### 7.8.1 🔬 引用计数（CPython 主要机制）

每个对象都有一个**引用计数器**。当：
- 创建对象 → 计数 +1
- 名字指向它 → +1
- 容器存它 → +1
- 名字解绑 / 容器删除 / 函数返回 → -1
- 计数归零 → 立即释放

```python
import sys

a = [1, 2, 3]
print(sys.getrefcount(a))    # 输出: 2  
# 注意：getrefcount 自身的参数也是一次引用，所以比直觉多 1

b = a
print(sys.getrefcount(a))    # 输出: 3

del b
print(sys.getrefcount(a))    # 输出: 2

a = None                      # 对象引用归零，被回收
```

**优点**：
- 简单直接
- 对象不再使用时立即释放（确定性回收）
- 析构函数（`__del__`）调用时机可预测

**缺点**：
- **循环引用会泄漏**

### 7.8.2 ⚠️ 循环引用问题

```python
# ─────────────────────────────────────
# 循环引用：仅靠引用计数无法回收
# ─────────────────────────────────────
class Node:
    def __init__(self, name):
        self.name = name
        self.partner = None
    def __del__(self):
        print(f"{self.name} 被回收")

a = Node("A")
b = Node("B")
a.partner = b      # A → B
b.partner = a      # B → A，形成循环

del a
del b
# 即使外部名字都删了，a 和 b 仍互相引用，引用计数都是 1
```

**内存示意**：

```
del 之前：
   名字 a ──┐
            ▼
       ┌────────┐ partner  ┌────────┐
       │ Node A │─────────►│ Node B │
       │        │◄─────────│        │
       └────────┘  partner └────────┘
            ▲
   名字 b ──┘

del a, del b 之后：
       ┌────────┐ partner  ┌────────┐
       │ Node A │─────────►│ Node B │
       │        │◄─────────│        │
       └────────┘  partner └────────┘
       
       引用计数都是 1（互相指着），永远不归零！
```

### 7.8.3 🔬 GC 模块（解决循环引用）

CPython 有第二套机制：**分代垃圾回收器**（gc 模块）。

工作原理（简化版）：
1. GC 维护所有"容器对象"（可能含其他对象的对象）
2. 定期扫描，找到**可达性**为零的对象组（即使内部互相引用，但外部没有引用进入）
3. 回收整个对象组

```python
import gc

# 手动触发 GC
collected = gc.collect()
print(f"回收了 {collected} 个对象")

# 查看 GC 统计
print(gc.get_count())   # 输出: (代0, 代1, 代2) 当前未回收数

# 关闭 GC（性能敏感场景）
gc.disable()
# ... 一段密集代码 ...
gc.enable()
gc.collect()
```

**分代策略**：
- 新对象在第 0 代，回收频率高
- 经过几次 GC 还活着的对象升级到第 1、2 代
- 高代回收频率低（"老对象大概率还会活着"的假设）

### 7.8.4 弱引用：不增加计数

```python
import weakref

class Cache:
    pass

c = Cache()
ref = weakref.ref(c)      # 创建弱引用，不增加引用计数

print(ref())              # 输出: <Cache object>  通过 ref() 获取对象
del c                     # 对象被回收（引用计数归零）
print(ref())              # 输出: None  弱引用不能阻止回收
```

**应用场景**：缓存、观察者模式、避免循环引用。

## 7.9 内存优化：interning 与缓存

### 7.9.1 字符串驻留（intern）

```python
# 短的、像标识符的字符串会被自动驻留
a = "hello"
b = "hello"
print(a is b)        # 输出: True

# 长字符串或含特殊字符不会自动驻留
a = "hello world!"
b = "hello world!"
print(a is b)        # 输出: 可能 False（取决于实现）

# 强制驻留
import sys
a = sys.intern("some long string")
b = sys.intern("some long string")
print(a is b)        # 输出: True
```

**应用**：高频使用的字符串（如字典 key）驻留可加速比较。

### 7.9.2 小整数缓存

```python
# CPython 启动时预创建 [-5, 256] 的整数对象
print([id(i) == id(j) for i, j in zip(range(-5, 257), range(-5, 257))])
# 全部 True
```

## 7.10 实际开发场景

### 场景 1：函数返回时清理大对象

```python
def process_huge_data():
    huge = load_huge_dataset()    # 占 1GB 内存
    result = analyze(huge)        # 计算结果（小）
    return result
    # 函数返回时，huge 引用计数归零，立即释放 1GB

# vs 全局变量持有
huge_global = load_huge_dataset()
def process():
    return analyze(huge_global)
# huge_global 直到程序退出才释放
```

**最佳实践**：大对象尽量局部化，依赖引用计数自动清理。

### 场景 2：避免意外共享

```python
# ❌ 共享了内层列表
matrix = [[0] * 3] * 3
matrix[0][0] = 1
print(matrix)
# 输出: [[1, 0, 0], [1, 0, 0], [1, 0, 0]]   ← 三行是同一个列表！

# ✅ 用列表推导生成独立的内层
matrix = [[0] * 3 for _ in range(3)]
matrix[0][0] = 1
print(matrix)
# 输出: [[1, 0, 0], [0, 0, 0], [0, 0, 0]]
```

### 场景 3：缓存 vs 内存

```python
from functools import lru_cache

# lru_cache 持有对象引用，可能阻止回收
@lru_cache(maxsize=None)   # 无限缓存可能内存泄漏！
def slow_func(arg):
    return expensive(arg)

# 设置上限
@lru_cache(maxsize=128)    # 最多缓存 128 个
def safer(arg):
    return expensive(arg)
```

## 7.12 面试常问

### Q1：Python 是按值传递还是按引用传递？

**答**：都不准确。Python 是**按对象引用传递**（pass by object reference）。
- 函数参数和实参指向同一对象
- 在函数内重新赋值参数 → 不影响外部
- 在函数内修改可变对象 → 外部可见
- 不可变对象天然"看起来像"按值传递

### Q2：`is` 和 `==` 的区别？什么时候必须用 `is`？

**答**：
- `==` 比较值（调用 `__eq__`）
- `is` 比较身份（同一对象）
- 与 `None`、`True`、`False`、单例对象比较时用 `is`

### Q3：默认参数为什么不能用可变对象？

**答**：默认参数在**函数定义时**创建一次，存在 `func.__defaults__`。每次调用如果没传该参数，会复用同一对象。可变对象会累积修改。

### Q4：Python 的垃圾回收机制？

**答**：
1. **主要机制：引用计数**——对象引用归零立即回收
2. **辅助机制：分代 GC**——清理循环引用
3. **三代分代**：新对象在第 0 代，存活越久代越高，高代扫描少

### Q5：浅拷贝和深拷贝的区别？

**答**：
- 浅拷贝（`copy()` / `[:]` / `copy.copy`）：复制一层，内层共享
- 深拷贝（`copy.deepcopy`）：递归复制所有层
- 深拷贝有性能开销，能浅尽量浅

### Q6：以下代码输出？

```python
def f(lst=[]):
    lst.append(1)
    return lst

print(f())   # ?
print(f())   # ?
print(f([])) # ?
print(f())   # ?
```

**答**：
```
[1]
[1, 1]      ← 默认列表累积了
[1]         ← 显式传新列表，不影响默认值
[1, 1, 1]   ← 默认列表又累积一次
```

### 练习 7.4（高级）

以下代码有内存泄漏吗？为什么？怎么修？

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.next = None

def create_cycle():
    a = Node(1)
    b = Node(2)
    a.next = b
    b.next = a
    # 函数返回，a/b 局部名字消失
    # 但 a.next 和 b.next 互指
```

<details>
<summary>答案</summary>

短期会"看起来"泄漏，但 Python 的 **GC 模块**会清理循环引用，所以最终会被回收。

但有两个改进方法：

1. **手动断链**：
```python
def create_cycle():
    a = Node(1)
    b = Node(2)
    a.next = b
    b.next = a
    a.next = None   # 显式断链
    b.next = None
```

2. **用弱引用**：
```python
import weakref

class Node:
    def __init__(self, value):
        self.value = value
        self._next = None
    
    @property
    def next(self):
        return self._next() if self._next else None
    
    @next.setter
    def next(self, node):
        self._next = weakref.ref(node) if node else None
```

注意：如果 `Node` 定义了 `__del__`，老版本 Python（3.4 之前）的 GC 不能回收循环引用中含 `__del__` 的对象。3.4+ 修复了这个问题。
</details>

### 练习 7.6（综合）

实现一个安全的 `freeze` 函数，将任意嵌套结构转为不可变对象（用于做字典 key）：

```python
def freeze(obj):
    # ?
    pass

# 测试
freeze([1, 2, 3])              # → (1, 2, 3)
freeze({1, 2})                 # → frozenset({1, 2})
freeze({"a": 1, "b": [1, 2]})  # → frozenset({("a", 1), ("b", (1, 2))})
freeze([[1, 2], [3, 4]])       # → ((1, 2), (3, 4))
```

<details>
<summary>答案</summary>

```python
def freeze(obj):
    if isinstance(obj, dict):
        return frozenset((k, freeze(v)) for k, v in obj.items())
    if isinstance(obj, (list, tuple)):
        return tuple(freeze(x) for x in obj)
    if isinstance(obj, set):
        return frozenset(freeze(x) for x in obj)
    return obj   # 不可变对象直接返回

# 测试
print(freeze([1, [2, 3], 4]))
# (1, (2, 3), 4)

# 现在可以做 key
d = {freeze({"a": 1}): "value"}
```
</details>

## 本章总结

|------|------------|
| 一切皆对象 | 整数、函数、类都是对象，有 id/type/value |

| 引用计数 | 主回收机制，归零即销毁 |
