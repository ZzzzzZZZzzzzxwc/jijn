# 14. 性能优化

> **过早优化是万恶之源 —— 但能识别瓶颈是工程师的必备能力。**  
> 本章按"测量 → 优化 → 验证"的流程展开。

## 14.0 知识地图

```
性能优化方法论
1. 测量：profiling 工具找瓶颈（不要猜！）
2. 选型：换数据结构、换算法
3. 减少 Python 开销：本地化、内置函数、避免反射
4. 用 C 加速：NumPy / Cython / Numba
5. 并发：见 11、12 章
6. 缓存：lru_cache / Redis
```

## 14.1 测量优先

### 14.1.1 timeit：微基准测试

```python
import timeit

# 比较两种写法
t1 = timeit.timeit("[i*2 for i in range(1000)]", number=10000)
t2 = timeit.timeit("list(map(lambda i: i*2, range(1000)))", number=10000)
print(f"列表推导: {t1:.3f}s")
print(f"map+lambda: {t2:.3f}s")
```

命令行：
```bash
python -m timeit "sum(range(100))"
```

### 14.1.2 cProfile：函数级分析

```python
import cProfile

def my_app():
    # ... 你的代码 ...
    pass

cProfile.run("my_app()", sort="cumulative")
```

输出（节选）：
```
        100 function calls in 0.234 seconds
   ncalls  tottime  percall  cumtime  percall filename:lineno(function)
        1    0.000    0.000    0.234    0.234 <string>:1(<module>)
       10    0.200    0.020    0.230    0.023 mymod.py:5(slow_func)
```

字段：
- **ncalls**：调用次数
- **tottime**：函数自身耗时（不含子调用）
- **cumtime**：含子调用的总耗时

## 14.2 数据结构与算法选择

### 14.2.1 ★ 选对数据结构

| 操作 | list | dict | set | deque |
|------|------|------|-----|-------|
| 末尾追加 | O(1) | - | - | O(1) |
| 头部插入 | O(n) | - | - | O(1) |
| 按索引访问 | O(1) | - | - | O(n) |
| 按 key 查找 | O(n) | O(1) | O(1) | O(n) |
| in 判断 | O(n) | O(1) | O(1) | O(n) |

**实例**：

```python
# ❌ list 中频繁 in 检查 → O(n)
def has_duplicates_slow(items):
    seen = []
    for x in items:
        if x in seen:           # O(n)
            return True
        seen.append(x)
    return False
# 总复杂度 O(n²)

# ✅ set
def has_duplicates_fast(items):
    seen = set()
    for x in items:
        if x in seen:           # O(1)
            return True
        seen.add(x)
    return False
# 总复杂度 O(n)
```

### 14.2.2 内存与速度的权衡

```python
# 计数：用 Counter
from collections import Counter
c = Counter(words)              # 比手动 dict 加 if 快

# 默认值字典：用 defaultdict
from collections import defaultdict
d = defaultdict(list)
d["k"].append(1)               # 比 `if k not in d: d[k]=[]` 快且简洁

# 双端队列
from collections import deque
q = deque(maxlen=100)          # 自动丢弃旧元素
```

## 14.3 减少 Python 开销

### 14.3.1 ★ 全局名字本地化

```python
import math

# ❌ 每次 LOAD_GLOBAL
def slow():
    return [math.sqrt(i) for i in range(10**6)]

# ✅ 本地化
def fast():
    sqrt = math.sqrt
    return [sqrt(i) for i in range(10**6)]
```

实测 `fast` 快 10-20%。

### 14.3.2 用内置函数

```python
# ❌
total = 0
for x in items:
    total += x

# ✅ sum 是 C 实现
total = sum(items)
```

```python
# ❌
result = []
for x in items:
    result.append(x * 2)

# ✅ 列表推导更快
result = [x * 2 for x in items]

# ✅✅ 如果只需要迭代，用生成器表达式（省内存）
sum(x * 2 for x in items)
```

### 14.3.3 字符串拼接

```python
# ❌ O(n²)：每次创建新字符串
s = ""
for piece in pieces:
    s += piece

# ✅ O(n)
s = "".join(pieces)
```

### 14.3.4 减少属性访问

```python
# ❌
for _ in range(10**6):
    self.method()

# ✅
m = self.method
for _ in range(10**6):
    m()
```

### 14.3.5 ⚠️ 不要过度优化

可读性 > 微优化。只对**热点路径**优化（profile 已证实）。

## 14.4 NumPy / Pandas：向量化

### 14.4.1 ★ 向量化代替循环

```python
import numpy as np

a = np.arange(10**7)

# ❌ Python 循环 (~3s)
b = [x * 2 for x in a]

# ✅ NumPy 向量化 (~30ms)
b = a * 2
```

向量化的本质：在 C 层面循环，**释放 GIL**。

### 14.4.2 避免循环 DataFrame

```python
import pandas as pd

df = pd.DataFrame({"a": range(10**6), "b": range(10**6)})

# ❌ 慢
df["c"] = df.apply(lambda r: r["a"] + r["b"], axis=1)

# ✅ 快几百倍
df["c"] = df["a"] + df["b"]
```

## 14.5 缓存

### 14.5.1 ★ functools.lru_cache

```python
from functools import lru_cache

@lru_cache(maxsize=1000)
def expensive(arg):
    # 假装慢
    return arg ** 2

# 第一次调用真算，之后直接查表
expensive(10); expensive(10); expensive(10)
print(expensive.cache_info())   # CacheInfo(hits=2, misses=1, ...)
```

⚠️ 参数必须可哈希（不能是 list/dict）。

### 14.5.2 cached_property

```python
from functools import cached_property

class Data:
    @cached_property
    def heavy_computation(self):
        return sum(range(10**7))

d = Data()
d.heavy_computation     # 算一次
d.heavy_computation     # 直接返回缓存
```

### 14.5.3 外部缓存

```python
# Redis
import redis
r = redis.Redis()

def cached_fetch(key):
    cached = r.get(key)
    if cached:
        return cached.decode()
    value = expensive_fetch(key)
    r.setex(key, 3600, value)
    return value
```

## 14.6 用 C 扩展加速

### 14.6.1 Numba（最简单）

```python
from numba import njit

@njit
def slow_sum(arr):
    total = 0
    for x in arr:
        total += x
    return total

import numpy as np
arr = np.arange(10**7)
slow_sum(arr)    # 第一次有编译开销，之后接近 C 速度
```

### 14.6.2 Cython

```cython
# fast.pyx
def fast_sum(int n):
    cdef int total = 0
    cdef int i
    for i in range(n):
        total += i
    return total
```

```bash
cythonize -i fast.pyx
```

### 14.6.3 直接写 C 扩展

复杂度高，通常用上述工具替代。

### 14.6.4 用 PyO3（Rust）

```rust
// 用 Rust 写 Python 扩展
use pyo3::prelude::*;

#[pyfunction]
fn fast_sum(n: u64) -> u64 {
    (0..n).sum()
}
```

## 14.8 内存优化

### 14.8.1 用 generator 代替 list

```python
# ❌ 全部加载到内存
data = [process(line) for line in open("huge.txt")]

# ✅ 流式处理
data = (process(line) for line in open("huge.txt"))
for item in data:
    ...
```

### 14.8.2 __slots__：固定属性的类

```python
class Point:
    __slots__ = ("x", "y")    # 没有 __dict__
    def __init__(self, x, y):
        self.x = x
        self.y = y

# 节省内存：~30-40%
# 缺点：不能动态加属性
```

### 14.8.3 array 代替 list

```python
import array

# 存大量同类型数字
a = array.array("i", range(10**6))    # int 数组
# 比 list 省一半内存
```

### 14.8.4 NumPy 数组

```python
import numpy as np
a = np.zeros(10**6, dtype=np.int32)
# 比 list 省 5-10 倍内存
```

## 14.9 实战：优化案例

### 案例：日志聚合

```python
# 原始版（30 秒）
def aggregate_logs_v1(filename):
    counts = {}
    f = open(filename)
    lines = f.readlines()        # 一次读全部
    f.close()
    for line in lines:
        ip = line.split()[0]
        if ip in counts:
            counts[ip] += 1
        else:
            counts[ip] = 1
    return counts

# 优化版（5 秒）
from collections import Counter

def aggregate_logs_v2(filename):
    with open(filename) as f:
        return Counter(line.split(maxsplit=1)[0] for line in f)
```

改进点：
1. `with` 自动关闭
2. 流式处理（不读全部）
3. 用 Counter（C 实现）
4. `split(maxsplit=1)` 避免无谓分割
5. 生成器避免中间列表

## 14.10 常见误区

### 误区 1：以为 PyPy 万能

PyPy 是 JIT 实现的 Python，纯 Python 代码可快 5-10 倍。但与 C 扩展兼容性差，启动慢。

### 误区 2：过早微优化

```python
# 优化前先 profile
# 否则可能优化错地方
```

### 误区 3：过度使用多线程

CPU 密集任务多线程不快（GIL）。

### 误区 4：把 python 当 C 写

```python
# ❌ 风格不像 Python
i = 0
while i < len(items):
    if items[i] > 0:
        # ...
    i += 1

# ✅ 更快也更可读
for item in items:
    if item > 0:
        # ...
```

## 14.11 面试常问

### Q1：怎么找性能瓶颈？

**答**：
1. 先用 cProfile 找慢函数
2. 再用 line_profiler 定位慢行
3. 用 timeit 验证优化效果

### Q2：列表和元组哪个更快？

**答**：元组创建快、占内存少（不可变可优化），list 增删快。频繁创建短序列用元组。

### Q3：lru_cache 的限制？

**答**：
- 参数必须可哈希
- 缓存所有参数组合，可能内存膨胀
- 不感知数据更新（chache 不会自动失效）

### Q4：如何减少 Python 启动时间？

**答**：
- 减少 import（特别是慢的库放函数内 lazy import）
- 用 `python -S` 跳过 site
- 使用 `-O` 关闭 assert 和 docstring

## 14.12 练习题

### 练习 14.1（基础）

下面代码慢在哪？怎么优化？

```python
def find_common(a, b):
    common = []
    for x in a:
        if x in b and x not in common:
            common.append(x)
    return common
```

<details>
<summary>答案</summary>

`x in b`（list）是 O(n)，`x not in common` 也是 O(n)，总复杂度 O(n²) 甚至 O(n³)。

```python
def find_common(a, b):
    return list(set(a) & set(b))
# O(n+m)
```
</details>

### 练习 14.2（中级）

profile 以下代码并优化：

```python
def stats(data):
    avg = sum(data) / len(data)
    var = sum((x - avg) ** 2 for x in data) / len(data)
    return avg, var ** 0.5
```

<details>
<summary>答案</summary>

如果 data 大，用 NumPy：

```python
import numpy as np

def stats(data):
    arr = np.asarray(data)
    return float(arr.mean()), float(arr.std())
```

或用 statistics 模块（数值稳定性更好）：
```python
import statistics
statistics.mean(data), statistics.stdev(data)
```
</details>

## 本章总结

| 优化方向 | 工具/方法 |
|---------|-----------|
| 找瓶颈 | cProfile / line_profiler / tracemalloc |
