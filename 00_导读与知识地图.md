## 0.5 ⚠️ 规范 vs 实现

本文档严格区分**两类知识**：

### Python 语言规范（PEP / 官方文档）

跨所有 Python 实现都成立：
- 语法（async/await、yield、装饰器、解构...）
- 对象模型（一切皆对象、特殊方法协议）
- import 语义、迭代器协议、上下文协议
- typing 注解语法

### CPython 实现细节（🔬 标注）

**只在 CPython 上成立**，PyPy/Jython/MicroPython 可能不同：

- **GIL** — 不在语言规范中，PyPy 也有，Jython 没有
- **引用计数** — CPython 主回收机制，其他实现可能纯 GC
- **小整数缓存** `[-5, 256]`、字符串驻留 — CPython 优化
- **字节码指令**（LOAD_FAST 等）— CPython 实现细节，每版本可变
- **__slots__ 内存布局** — CPython 实现
- **dict 保持插入顺序** — 3.7+ 进入语言规范，之前是 CPython 实现
- **PEP 659 自适应解释器** — 3.11+ CPython 优化

→ 写跨实现兼容代码时务必区分。

## 0.6 配套学习方法

### 方法 1：手动追踪变量

读到代码不要直接看输出。在纸上或注释里画出：
- 每一步内存中有哪些对象
- 每个名字指向哪里
- 引用计数变化

例如：
```python
x = [1, 2]
y = x
y = y + [3]   # ← 这里 y 还指向原列表吗？
print(x)      # ← 你能预测输出吗？
```

正确：`y = y + [3]` 创建新列表，`y` 指向新列表，`x` 仍指向 `[1,2]`。输出 `[1, 2]`。  
不能瞬间答对 → 精读 [07 章](07_引用与内存机制.md)。

### 方法 2：用 dis 模块看字节码

```python
import dis
def f(x):
    return x + 1
dis.dis(f)
```

```
  2           0 RESUME                   0
  3           2 LOAD_FAST                0 (x)
              4 LOAD_CONST               1 (1)
              6 BINARY_OP                0 (+)
             10 RETURN_VALUE
```

每当不确定 Python 在干什么时，`dis.dis(func)` 是终极工具。详见 [10 章](10_GIL与CPython执行机制.md) 与 [20 章](20_AST与字节码深入.md)。

### 方法 3：用 id() 验证引用

```python
a = 256
b = 256
print(id(a), id(b), a is b)   # 同 id, True (CPython 小整数缓存)

a = 257
b = 257
print(id(a), id(b), a is b)   # 不同 id, False
```

`is` 比较身份（地址），`==` 比较值。详见 [07 章](07_引用与内存机制.md)。
