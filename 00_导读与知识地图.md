## 0.4 文档约定

### 代码块约定

```python
# ─────────────────────────────────────
# 示例：变量赋值过程追踪
# ─────────────────────────────────────
a = [1, 2, 3]      # 创建列表对象，a 指向它       内存: [1,2,3] @ 0x100
b = a              # b 指向同一个列表             内存: [1,2,3] @ 0x100 ← a, b
b.append(4)        # 修改这个列表对象             内存: [1,2,3,4] @ 0x100

print(a)           # 输出: [1, 2, 3, 4]   ← a 看到的也变了（同一对象）
print(a is b)      # 输出: True            ← 同一对象
```

每段代码遵循：
1. 顶部注释说明示例目的
2. 行内注释解释变量变化
3. `# 输出: ...` 标注实际运行结果
4. 关键步骤标注内存示意

## 0.5 配套学习方法

### 方法 1：手动追踪变量

读到一段代码时，**不要直接看输出**。在纸上或注释里画出：
- 每一步内存中有哪些对象
- 每个名字（变量名）指向哪里
- 引用计数变化（如果相关）

例如：
```python
x = [1, 2]
y = x
y = y + [3]   # ← 这里 y 还指向原列表吗？
print(x)      # ← 你能预测输出吗？
```

正确答案：`y = y + [3]` 创建新列表，`y` 指向新列表，`x` 仍指向 `[1,2]`。输出 `[1, 2]`。
如果不能瞬间答对，请精读 07 章。

### 方法 2：用 dis 模块看字节码

```python
import dis
def f(x):
    return x + 1
dis.dis(f)
```

```
  2           0 LOAD_FAST                0 (x)        # 把 x 推入栈
              2 LOAD_CONST               1 (1)        # 把 1 推入栈
              4 BINARY_ADD                            # 弹出两个，相加，结果入栈
              6 RETURN_VALUE                          # 弹出栈顶并返回
```

每当你不确定 Python 在干什么时，`dis.dis(func)` 是终极工具。10 章会详细讲。

### 方法 3：用 id() 验证引用

```python
a = 256
b = 256
print(id(a), id(b), a is b)   # 输出: 同一个 id, True  (小整数缓存)

a = 257
b = 257
print(id(a), id(b), a is b)   # 输出: 不同 id, False  (大整数不缓存)
```

**结论**：`is` 比较的是身份（内存地址），`==` 比较的是值。详见 07 章。

## 0.6 给读者的话

Python 看起来简单，但简单的语法下隐藏着复杂的运行时机制：
- 一个普通的 `a = b` 到底做了什么？
- 为什么 `def f(x=[]):` 是个陷阱？
- 为什么多线程跑不快？
- 为什么 `await` 不阻塞？

这些问题，速查文档不会告诉你。这份文档会。

如果你只想"能写出来"，看任何速成教程都行。
如果你想"知道自己在写什么"，请认真读完这份文档，并完成所有练习。
